# 功能演示流程

本文用于演示项目核心闭环。它不是完整测试用例，正式上线检查见 `上线验收清单.md`。

## 演示目标

展示项目已经具备真实低代码编辑器的基础能力：

```text
注册/登录 -> 创建项目 -> 创建页面 -> 编辑页面
  -> 保存恢复 -> 版本回滚 -> 发布公开访问
```

## 演示前准备

```bash
docker compose up -d postgres
npm run dev --prefix server
npm run dev
```

默认地址：

```text
前端：http://localhost:5173
后端：http://localhost:3000/api
```

建议先运行：

```bash
npm run check
```

如需验证 API 主链路：

```bash
npm run smoke:api
```

## 演示步骤

| 步骤 | 操作 | 说明重点 |
| --- | --- | --- |
| 1 | 注册或登录 | JWT 登录态，后续接口自动带 token |
| 2 | 创建项目 | 项目是页面和成员权限的归属 |
| 3 | 创建页面 | 后端写入默认 Page schema |
| 4 | 进入编辑器 | 页面 schema 迁移后进入 Zustand store |
| 5 | 拖拽物料 | 左侧物料 -> 中间画布，形成组件树 |
| 6 | 配置属性/样式 | 右侧设置面板写入组件 props/styles |
| 7 | 配置事件 | 事件动作写入 `props.onEvent` |
| 8 | 切换预览 | 使用 prod 物料和 Preview 运行态 |
| 9 | 保存页面 | 校验 schema 后保存到 PostgreSQL |
| 10 | 刷新恢复 | 重新读取后端 schema 并恢复组件树 |
| 11 | 查看版本 | 每次保存生成 PageVersion |
| 12 | 回滚版本 | 用历史快照恢复 Page.schema |
| 13 | 发布页面 | 生成 publish 快照和 publicId |
| 14 | 公开访问 | 访客无需登录访问 `/publish/:publicId` |

## 推荐演示页面内容

- Page 下拖入 Container 或 Card。
- 添加 Text、Button、Input、Select、Table、Alert。
- 给 Button 配置 toast 或 url 动作。
- 修改 Text 文案和样式。
- 保存后刷新，确认内容恢复。
- 发布后复制公开链接访问。

## 演示时可强调的技术点

- 编辑器本质是维护一棵组件树。
- dev 物料用于编辑态，prod 物料用于预览和发布态。
- 组件树保存为 Page.schema，版本保存为 PageVersion.schema。
- 发布页读取发布快照，不读取正在编辑的草稿。
- schema migration 兼容旧数据。
- 权限由后端校验，不依赖前端隐藏按钮。
- 公开发布页禁用 customJS。

## 常见演示风险

- 数据库未启动：后端无法连接。
- 后端未启动：前端 API 请求失败。
- `.env` API 地址错误：前端请求到错误后端。
- 页面 schema 非法：保存或发布会失败。
- 使用 viewer 账号：无法保存或发布。
