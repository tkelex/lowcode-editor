# 上线验收清单

## 使用方式

这份清单用于正式上线前的最后一轮验收。建议在接近生产的环境中执行，例如测试域名、生产同款 Docker Compose、独立数据库。

每个验收项都要记录结果。失败项修复后重新验收，不只重测失败项，也要重测相关链路。

## 1. 自动化检查

在项目根目录执行：

```powershell
npm.cmd run preflight
```

预期：

- [ ] lint 通过。
- [ ] 前端生产构建通过。
- [ ] 后端构建通过。
- [ ] 单元测试通过。
- [ ] mock E2E 通过。

真实 API smoke：

```powershell
npm.cmd run smoke:api
```

预期：

- [ ] 注册、登录通过。
- [ ] 项目、成员、页面、版本、发布、公开读取通过。
- [ ] 管理员接口、禁用用户、禁用项目、取消发布通过。
- [ ] 素材上传和公开读取通过。

## 2. 部署环境检查

- [ ] 域名能打开前端首页。
- [ ] `/api/health` 返回成功。
- [ ] 浏览器控制台没有持续 API 404。
- [ ] 后端日志没有启动异常。
- [ ] 数据库 migration 已执行。
- [ ] 上传目录或 volume 已挂载。

## 3. 用户主链路

### 注册和登录

- [ ] 新用户可以注册。
- [ ] 已有用户可以登录。
- [ ] 账号或密码错误时不会泄露用户是否存在。
- [ ] 刷新页面后仍保持登录态。

### 项目和页面

- [ ] 可以创建项目。
- [ ] 可以修改项目名称和描述。
- [ ] 可以创建页面。
- [ ] 页面路径重复时有错误提示。
- [ ] 可以打开编辑器。

### 编辑器

- [ ] 物料区能搜索组件。
- [ ] 能拖拽 Text/Button/Image/Card/Input/Select/Switch/Form/Table 等常用组件。
- [ ] 能修改组件属性。
- [ ] 能修改组件样式。
- [ ] 能绑定基础事件动作。
- [ ] 预览模式能正常打开和返回。
- [ ] 源码面板能展示当前 schema。
- [ ] 非法源码不能应用。

### 保存和恢复

- [ ] 保存页面成功。
- [ ] 保存后生成版本记录。
- [ ] 刷新或重新打开页面后组件树恢复。
- [ ] 回滚旧版本后画布恢复。
- [ ] 删除历史版本不影响当前画布。

### 发布

- [ ] 发布页面成功。
- [ ] 公开地址无需登录可访问。
- [ ] 发布页显示发布快照。
- [ ] 发布后继续编辑草稿，不影响已发布快照。
- [ ] 取消发布后公开地址返回 404。

## 4. 权限和协作

- [ ] owner 可以管理项目成员。
- [ ] editor 可以编辑页面但不能管理成员。
- [ ] viewer 只能查看，不能保存或发布。
- [ ] 非项目成员不能访问项目页面。
- [ ] 项目审计日志只允许 owner 查看。

## 5. 管理员后台

- [ ] 普通用户看不到“管理后台”入口。
- [ ] 普通用户直接访问 `/admin` 有无权限提示。
- [ ] 非管理员访问 `/api/admin/overview` 返回 403。
- [ ] 管理员可以进入 `/admin`。
- [ ] 管理员总览数据正常。
- [ ] 用户管理可以搜索和筛选。
- [ ] 管理员不能禁用当前登录账号。
- [ ] 禁用用户后，该用户旧 token 失效。
- [ ] 项目管理可以搜索和筛选。
- [ ] 禁用项目后普通用户不能访问该项目。
- [ ] 禁用项目后公开页面返回 404。
- [ ] 发布页列表可以取消发布。
- [ ] 全局审计日志能看到管理员操作。

## 6. 素材上传

- [ ] 合法图片可以上传。
- [ ] 合法文本或 PDF 可以上传。
- [ ] 非法类型被拒绝。
- [ ] 超大文件被拒绝。
- [ ] 项目 viewer 不能上传素材。
- [ ] 禁用项目后公开素材读取返回 404。

## 7. 生产安全

- [ ] 生产环境不是示例 `JWT_SECRET`。
- [ ] 生产 `FRONTEND_ORIGIN` 不是 localhost。
- [ ] 生产 `VITE_LOWCODE_HTTP_ALLOWED_ORIGINS` 已收口。
- [ ] 公开发布页不执行 custom JS。
- [ ] 数据库端口没有暴露公网。
- [ ] `.env` 和备份文件没有进入 Git。

## 8. 备份和回滚

- [ ] 上线前已创建数据库备份。
- [ ] 备份文件能被找到。
- [ ] 恢复演练通过。
- [ ] 回滚流程已确认。
- [ ] 管理员账号授予流程已确认。

## 9. 验收结论

上线前填写：

```text
验收环境：
验收时间：
验收人：
Git commit：
数据库备份文件：
是否通过：
遗留问题：
```

只有“是否通过”为“是”，且没有阻断项时，才建议开放给真实用户试用。
