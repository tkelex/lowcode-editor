# 上线安全检查清单

## 使用方式

每次准备把项目部署到公网或给真实用户试用前，按本清单逐项确认。不能确认的项目不要跳过，应记录原因和处理人。

## 环境变量

- [ ] `NODE_ENV=production`。
- [ ] `JWT_SECRET` 已替换为至少 32 位随机强密钥。
- [ ] `JWT_SECRET` 不是示例值、默认值或曾提交到 Git 的值。
- [ ] `FRONTEND_ORIGIN` 是真实前端域名，不是 `localhost`。
- [ ] `DATABASE_URL` 使用生产数据库地址和强密码。
- [ ] `UPLOAD_DIR` 指向持久化目录或 volume。
- [ ] `UPLOAD_MAX_SIZE` 符合预期。
- [ ] `VITE_API_BASE_URL` 是 `/api` 或可信 API 域名。
- [ ] `VITE_LOWCODE_HTTP_ALLOWED_ORIGINS` 已按业务需要收口，不为空泛通配。

## 数据库

- [ ] 生产数据库没有暴露公网。
- [ ] 数据库账号不是超级用户。
- [ ] 上线前已执行数据库备份。
- [ ] 备份文件大小正常，不是 0。
- [ ] 至少完成过一次恢复演练。
- [ ] migration 通过 `npm.cmd run prisma:deploy --prefix server` 执行。

## 鉴权和权限

- [ ] 注册用户默认为普通用户。
- [ ] 管理员只能通过脚本或内部流程授予。
- [ ] 至少有一个可用管理员账号。
- [ ] 非管理员访问 `/api/admin/*` 返回 403。
- [ ] 禁用用户后，旧 token 访问 `/api/auth/me` 返回 `AUTH_ACCOUNT_DISABLED`。
- [ ] 禁用项目后，普通用户不能读取、编辑、保存、发布该项目。
- [ ] 禁用项目后，公开发布页返回 404。
- [ ] 禁用项目后，公开素材读取返回 404。

## 发布页安全

- [ ] 公开发布页使用 `publicId`，不暴露自增 `pageId`。
- [ ] 公开接口只返回公开展示需要的字段。
- [ ] 公开发布页读取发布快照，不读取编辑草稿。
- [ ] 公开运行态传入 `allowCustomJS={false}`。
- [ ] 取消发布后公开地址返回 404。

## 上传安全

- [ ] 上传文件类型白名单有效。
- [ ] 上传大小限制有效。
- [ ] 上传路径不能逃逸 `UPLOAD_DIR`。
- [ ] 上传目录已持久化。
- [ ] 备份策略已覆盖素材元数据；如本地 volume 存储文件，也需要同步备份上传目录或 volume。

## 前端运行态安全

- [ ] 生产环境 HTTP 动作域名白名单已配置。
- [ ] 不把服务端 token、数据库连接串或密钥注入低代码动作配置。
- [ ] 编辑器预览中的 custom JS 不会在公开发布页执行。
- [ ] 发布页渲染异常有兜底，不会让用户卡在空白页。

## Git 和日志

- [ ] `.env`、真实数据库密码、JWT secret 没有进入 Git。
- [ ] 数据库备份文件没有进入 Git。
- [ ] 日志不输出密码、token、数据库连接串。
- [ ] 生产日志能定位 method、path、status、duration 或至少能定位错误接口。

## 上线前命令

本地质量检查：

```powershell
npm.cmd run preflight
```

真实后端接口检查：

```powershell
npm.cmd run smoke:api
```

生产 compose 配置检查：

```powershell
docker compose --env-file infra/docker/.env -f infra/docker/docker-compose.prod.example.yml config --quiet
```

## 阻断上线项

以下任一项不满足，应阻断上线：

- 使用弱 `JWT_SECRET`。
- 没有数据库备份。
- 无法恢复数据库。
- 非管理员可以访问管理员接口。
- 公开发布页能执行自定义 JS。
- 数据库暴露公网且没有强认证。
- 生产环境仍使用 localhost 作为 `FRONTEND_ORIGIN`。
