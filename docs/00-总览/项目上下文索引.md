# 项目上下文索引

这是新协作者和新 AI 对话的低 token 项目入口。它只记录稳定事实、阅读路线和当前验证状态；具体实现以源码和专题文档为准。

## 一句话说明

这是一个 Vite + React + TypeScript + NestJS + PostgreSQL 的低代码编辑器。用户可以登录后创建项目和页面，在编辑器中拖拽物料、配置属性/样式/事件，保存 schema，管理版本，回滚，并发布公开访问页面。

## 技术栈

| 层 | 技术 |
| --- | --- |
| 前端 | Vite、React、TypeScript |
| UI | Ant Design、Tailwind CSS、Allotment |
| 编辑器状态 | Zustand persist |
| 拖拽 | React DnD |
| 代码编辑 | Monaco Editor |
| 后端 | NestJS、TypeScript |
| 数据库 | PostgreSQL |
| ORM | Prisma |
| 鉴权 | JWT、Passport |
| 共享 schema | `packages/lowcode-schema` |

## 核心业务闭环

```text
注册 / 登录
  -> 创建项目
  -> 创建页面
  -> 打开编辑器
  -> 读取并迁移 Page.schema
  -> 写入 Zustand 组件树
  -> 拖拽物料、配置属性/样式/事件
  -> 保存前校验 schema
  -> PATCH /api/pages/:id
  -> 后端迁移、校验并保存 Page.schema
  -> 创建 PageVersion
  -> 发布生成 publish 快照
  -> 访客访问 /publish/:publicId
```

## 关键约定

- 组件树根节点是 `Page`，通常 id 为 `1`。
- 组件节点结构：`{ id, name, props, styles, desc, children?, parentId? }`。
- 主 store 导出名保留历史拼写：`useComponetsStore`。
- 新拖拽 hook 使用 `useMaterialDrop`，历史拼写 `useMaterailDrop` 保留兼容。
- 新 Preview 路径使用 `components/Preview` 或 `runtime/Preview`，历史拼写 `components/Preivew` 保留兼容。
- 新源码面板路径使用 `components/Source`，历史拼写 `components/Sourse` 保留兼容。
- 公开发布页必须传 `allowCustomJS={false}`。

## 高风险区域

- `src/editor/stores/components.tsx`：组件树、选择、撤销重做、本地缓存。
- `src/editor/registry/component-config.tsx`：物料、setter、事件、方法、dev/prod 映射。
- `src/editor/runtime/Preview/index.tsx`：事件动作、运行态变量、数据源、自定义 JS。
- `packages/lowcode-schema/src/*`：前后端 schema 迁移和校验。
- `server/src/modules/projects/project-access.service.ts`：项目成员权限。
- `server/src/modules/pages/*`：页面保存、版本、发布、公开读取。
- `server/prisma/schema.prisma`：数据库模型，改动必须配套 migration。
