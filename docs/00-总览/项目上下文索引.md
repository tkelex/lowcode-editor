## 项目一句话说明

这是一个从 React 学习型 demo 演进中的低代码编辑器项目，目标是成为可上线的全栈产品。当前已具备注册登录、项目管理、项目 owner/editor/viewer 成员权限、项目成员管理、项目与页面操作审计、页面管理、编辑器 schema 保存到 PostgreSQL、重新读取恢复页面、页面版本历史、回滚、删除版本记录、页面发布公开访问、共享 schema 迁移与校验、源码面板 JSON 应用、保存/发布前 schema 校验、撤销/重做、响应式画布、右键菜单、大纲树定位/重命名/排序、分类物料区、搜索/收藏物料、常用模板、P3 基础/表单/数据/反馈物料、表单表格增强和常用物料事件绑定的核心闭环。

## 当前核心业务闭环

```text
注册 / 登录
  ↓
创建项目
  ↓
创建页面
  ↓
打开编辑器
  ↓
后端 Page.schema → migratePageSchema → 前端 Zustand components
  ↓
拖拽 Text / Image / Card / Alert / Input / Select / Switch / Grid / Form / Table / Chart 等分类物料、配置属性、样式、事件
  ↓
撤销 / 重做可回退组件树编辑操作
  ↓
点击保存
  ↓
前端统一校验组件树 schema
  ↓
前端 components → PATCH /api/pages/:id → PostgreSQL Page.schema
  ↓
后端创建 PageVersion
  ↓
版本历史可回滚到旧 schema，也可删除无用版本记录
  ↓
点击发布生成 PageVersion(source = publish) 快照
  ↓
访客无需登录访问 /publish/:publicId，公开页渲染发布快照
  ↓
再次打开页面恢复组件树
```

## 当前重要命令

根目录前端：

```bash
npm install
npm run dev
npm run build
npm run lint
npm run test
npm run check
npm run smoke:api
```

后端：

```bash
npm install --prefix server
npm run dev --prefix server
npm run build --prefix server
npm run prisma:generate --prefix server
npm run prisma:migrate --prefix server -- --name add_page_versions
```

本地 PostgreSQL：

```bash
docker compose up -d postgres
docker compose ps
docker compose down
```

如果当前 Bash 找不到 Docker，Windows 本机环境可使用：

```bash
"/c/Program Files/Docker/Docker/resources/bin/docker.exe" compose ps
```

## 必须保留的历史拼写

这些名字已经成为当前代码 API，除非做全量迁移，否则不要随手删除。当前正确命名入口已经存在，旧拼写保留为兼容导出：

- `useComponetsStore`
- `useMaterailDrop`
- `components/Preivew`
- `components/Sourse`
- 新代码优先使用 `useMaterialDrop`、`components/Preview`、`components/Source`

## 重要安全提醒

- 不要提交 `.env`、token、数据库密码或真实 JWT secret。
- 后端业务接口必须走 JWT Guard，公开发布页读取接口除外。
- 项目和页面接口必须校验 owner，不能跨用户访问。
- 预览模式支持 `customJS`，它会执行用户配置的 JS；公开发布页通过 `allowCustomJS={false}` 禁用该能力。
