# 物料体系说明

本文记录 P3 阶段完成的物料扩展和后续新增物料约定。

## 已内置物料

### 布局和基础

- 布局容器：Container、Card、Space、Flex、Grid。
- 基础组件：Button、Link、Icon、Text、Image、Divider、Tabs、Steps。

### 表单

- 表单容器：Form、FormItem。
- 独立表单控件：Input、Textarea、Select、Radio、Checkbox、DatePicker、Switch、Upload、Rate。
- FormItem 支持 input、textarea、date、select、radio、checkbox、switch、rate。
- Form 支持提交、重置、值变化事件，并暴露 submit、reset、getValues、setValues 方法。

### 数据展示

- Table、TableColumn。
- List、Descriptions、Statistic、Pagination、Chart。
- Table 支持静态 JSON 数据、远程 URL 数据源、分页、空状态、日期列和操作列。
- Chart 当前使用轻量 CSS 条形图实现，后续接入 ECharts 或 AntV 时可以保持同一 schema。

### 反馈

- Alert、Modal、Drawer、Tooltip、Popover、Notification、Result、Empty。
- Modal 和 Drawer 都暴露 open、close 方法，可通过事件动作里的组件联动打开或关闭。

## 常用模板

物料区模板已包含：

- 内容展示区。
- 基础表单。
- 数据卡片。
- 表单页。
- 详情页。
- 列表页。
- 弹窗表单。
- 仪表盘。

模板会生成完整组件树并补齐 parentId，仍然走统一保存、预览、发布和 schema 校验链路。

## 新增物料约定

1. 每个物料必须同时提供编辑态 dev 和运行态 prod。
2. 可接收子组件的物料必须接入 useMaterialDrop，并在前端 registry 和共享 schema registry 中同步 acceptsChildren。
3. 运行态 prod 必须过滤低代码内部字段，并正确转发 registry 声明的事件 props。
4. 物料必须在 `src/editor/registry/component-config.tsx` 中配置 category、icon、keywords、sort、setter、events、methods。
5. 新增事件必须同步 `docs/04-编辑器/事件能力矩阵.md`。
6. 涉及预览或发布渲染的复杂物料必须补充回归测试。
