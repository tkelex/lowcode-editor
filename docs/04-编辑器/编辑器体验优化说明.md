# 编辑器体验优化说明

本文记录 P2 阶段完成的编辑器画布和配置体验能力，作为后续继续扩展物料、配置项和快捷操作时的交互约定。

## 画布

- 画布支持桌面、平板、手机三种宽度切换：1080px、768px、390px。
- 组件 hover 和 selected 使用统一遮罩定位，跟随滚动、窗口 resize 和组件尺寸变化。
- 选中组件后浮层提供快捷操作：复制、上移、下移、包裹容器、删除。
- 画布右键菜单提供同样的结构操作，根页面不允许复制、删除、移动或包裹。
- 拖拽过程中画布会进入强调状态，容器类物料继续使用可投放、可接收、禁止投放三类状态。

## 右侧配置面板

- 面板顶部展示当前组件名称、组件类型和组件 id。
- 属性、样式、事件三个页签共用搜索入口。
- 属性配置分为基础信息和属性配置两组。
- 样式配置分为快捷样式和 CSS 源码两组。
- 事件配置支持按事件名、事件说明、事件数据和动作类型搜索。
- 搜索无结果时展示空状态，避免用户误以为配置丢失。

## 源码区

- 源码区支持格式化 JSON、复制源码、重置为当前画布、应用源码。
- 源码编辑时会即时校验 JSON 和组件树合法性。
- 应用源码前会展示变更摘要，确认后才替换整个画布组件树。
- 应用失败会保留当前编辑内容并显示错误，不直接覆盖画布。

## 物料区

- 物料区保留分类和搜索，并新增收藏视图。
- 收藏状态保存在浏览器 localStorage，便于个人常用物料快速访问。
- 常用模板支持一键添加到页面根节点，目前包含内容展示区、基础表单、数据卡片。
- 模板生成的组件树会补齐父子关系，继续走统一保存、预览和发布链路。

## 大纲树

- 大纲树支持点击定位画布组件。
- 大纲树支持组件重命名，实际更新组件 `desc`。
- 大纲树支持拖拽排序和拖入容器，拖拽时会校验目标组件是否允许接收该类型子组件。
- 大纲树选中状态和画布当前选中组件保持同步。

## 后续约定

- 新增物料时必须确认是否能被大纲拖拽、是否能被模板引用、是否需要右键菜单保护。
- 新增配置项时要补充搜索关键字和空状态，不要只堆在单一表单里。
- 新增模板时尽量使用已有内置物料，避免模板成为绕过物料规范的特殊入口。
