# 物料事件与动作规划

本文只保留事件系统的当前规划结论。实现设计见 `事件绑定系统设计.md`，具体支持矩阵见 `事件能力矩阵.md`。

## 目标

事件系统需要回答两个问题：

1. 每个物料能触发哪些事件。
2. 每个事件允许配置哪些动作，以及能拿到哪些事件数据。

统一存储结构：

```ts
component.props.onEvent[eventName].actions
```

旧结构如 `props.onClick.actions` 由 schema migration 兼容迁移。

## 当前物料分组

| 类型 | 物料 | 事件策略 |
| --- | --- | --- |
| 布局容器 | Page、Container、Card、Flex、Grid、Space | 以点击、容器生命周期和子组件承载为主 |
| 基础展示 | Text、Image、Divider、Icon、Link | 保留 click、doubleClick、error 等轻量事件 |
| 表单输入 | Input、Select、Switch、Form、FormItem | 重点支持 change、finish、valuesChange |
| 数据展示 | Table、TableColumn、List、Chart、Descriptions | 支持加载、分页、行点击、选择等数据相关事件 |
| 反馈 | Alert、Modal、Toast 类能力 | 支持 close、ok、cancel、openChange |

结构配置节点如 `FormItem`、`TableColumn` 通常不单独绑定事件，由父组件统一处理。

## 动作类型

| 动作 | 当前定位 |
| --- | --- |
| `toast` | 显示提示消息 |
| `url` | 跳转或打开链接 |
| `componentAction` | 调用目标组件方法 |
| `confirm` | 二次确认后执行子动作 |
| `condition` | 按表达式结果执行分支 |
| `http` | 发起 HTTP 请求并写入事件数据 |
| `setVariable` | 修改运行态页面变量 |
| `setComponentProps` | 修改运行态组件 props |
| `setComponentStyles` | 修改运行态组件 styles |
| `custom` | 编辑器预览中执行自定义 JS |

公开发布页必须禁用 `custom`。

## 事件数据约定

常见事件数据：

| 事件 | 典型数据 |
| --- | --- |
| `click` | 当前组件、原始事件参数 |
| `change` | `value`、当前组件、原始事件参数 |
| `finish` | `values` |
| `finishFailed` | `errorFields`、`values` |
| `valuesChange` | `changedValues`、`allValues` |
| `rowClick` | `record`、`index` |
| `loadSuccess` | `data` |
| `loadError` | `error` |

事件数据由 `src/editor/events/createEventData.ts` 生成，表达式和动作执行由 `src/editor/events/runtime.ts` 处理。

## 推荐优先级

### P0：稳定当前闭环

- 保持 Button、Text、Image、Card、Container、Alert、Input、Select、Switch、Form、Table 的常用事件可用。
- 保证动作弹窗编辑已有动作时不会丢配置。
- 保证 Preview 注入事件 props 后，prod 物料能正确转发。
- 保证公开页 `allowCustomJS=false`。

### P1：增强数据联动

- Table 远程数据加载事件。
- 表单提交结果写入变量。
- 组件属性绑定 `{{variables.xxx}}` 的体验优化。
- 数据源请求状态可视化。

### P2：高级编排

- 更完整的条件分支。
- 全局事件或页面生命周期。
- 动作执行日志和调试面板。
- 自定义 JS 沙箱化。

## 改事件系统时必须检查

- `src/editor/registry/component-config.tsx` 中是否声明事件。
- `src/editor/components/Setting/ComponentEvent.tsx` 是否能展示和编辑。
- `src/editor/components/Setting/ActionModal.tsx` 是否正确初始化已有动作。
- `src/editor/runtime/Preview/index.tsx` 是否注入对应 React/AntD 事件 prop。
- 对应 `materials/<Name>/prod.tsx` 是否转发事件 prop。
- `packages/lowcode-schema/src/migrate.ts` 是否需要兼容旧数据。
- `scripts/test/*.test.mjs` 是否需要补充覆盖。
