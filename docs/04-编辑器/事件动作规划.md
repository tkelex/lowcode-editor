# 物料事件与动作规划

本文只保留事件系统的当前规划结论。实现设计见 `事件绑定系统设计.md`，具体支持矩阵见 `事件能力矩阵.md`。

## 目标

事件系统需要回答两个问题：

1. 每个物料能触发哪些事件。
2. 每个事件允许配置哪些动作，以及能拿到哪些事件数据。

统一存储结构：

```ts
component.props.onEvent[eventName].actions
```

旧结构如 `props.onClick.actions` 由 schema migration 兼容迁移。

## 当前物料分组

| 类型 | 物料 | 事件策略 |
| --- | --- | --- |
| 布局容器 | Page、Container、Card、Flex、Grid、Space | Page 不直接绑定事件；Container、Card 等可交互容器以 `click`、`doubleClick` 为主 |
| 基础展示 | Button、Text、Image、Divider、Icon、Link、Tabs、Steps | 操作型组件保留 `click`；可切换组件使用 `change`；双击只保留在少量高价值组件上 |
| 表单输入 | Input、Textarea、Select、Radio、Checkbox、DatePicker、Switch、Upload、Rate、Form、FormItem | 输入组件重点支持 `change`、`blur`、`pressEnter`；Form 统一承载 `finish`、`finishFailed`、`valuesChange` |
| 数据展示 | Table、TableColumn、List、Chart、Descriptions、Statistic、Pagination | 当前保留 `click`、`change` 等基础事件；行点击、加载成功/失败等数据事件作为后续增强 |
| 反馈 | Alert、Modal、Drawer、Tooltip、Popover、Notification、Result、Empty | 弹层类组件支持 `close`、`ok`、`cancel`、`openChange`，普通反馈组件保留轻量点击事件 |

结构配置节点如 `FormItem`、`TableColumn` 通常不单独绑定事件，由父组件统一处理。

## 事件面板交互约定

右侧事件页签采用接近 amis editor 的“添加事件 -> 事件分组 -> 动作条 -> 动作配置”模型：

1. 顶部只保留一个主入口 `添加事件`，点击后展示当前组件 registry `events` 中尚未添加的事件。
2. 用户添加事件后，面板生成事件分组条；分组条展示事件名、事件说明、动作数量、添加动作、清空动作、删除事件、展开/收起。
3. 面板默认只展示已添加事件，不再把所有可声明事件铺开；没有事件时展示空状态，引导用户先添加事件。
4. 每个事件下面按 `props.onEvent[eventName].actions` 顺序展示动作条。
5. 动作条展示动作类型、动作摘要、启用状态，并提供排序、复制、禁用/启用、设置和删除。
6. 添加或编辑动作时使用统一动作配置弹窗；弹窗左侧按场景分类选择动作，右侧展示该动作专属配置表单。
7. 动作列表必须按当前事件 `allowedActions` 过滤，不能配置不被事件允许的动作。
8. 事件添加、动作添加、动作编辑都只写入 `props.onEvent[eventName].actions`，不改变 schema。

## 动作类型

| 动作 | 当前定位 |
| --- | --- |
| `toast` | 显示提示消息 |
| `url` | 跳转或打开链接 |
| `componentAction` | 调用目标组件方法 |
| `confirm` | 二次确认后执行子动作 |
| `condition` | 按表达式结果执行分支 |
| `http` | 发起 HTTP 请求并写入事件数据 |
| `setVariable` | 修改运行态页面变量 |
| `setComponentProps` | 修改运行态组件 props |
| `setComponentStyles` | 修改运行态组件 styles |
| `custom` | 编辑器预览中执行自定义 JS |

公开发布页必须禁用 `custom`。

## 动作配置分类

| 分类 | 动作 | 配置重点 |
| --- | --- | --- |
| 页面 | `url` | 页面地址、打开方式 |
| 弹窗消息 | `toast`、`confirm` | 消息类型、文案、确认/取消分支 |
| 服务 | `http` | 请求地址、方法、鉴权、headers/body、响应/错误映射 |
| 组件联动 | `componentControl`、`componentAction`、`setComponentProps`、`setComponentStyles` | 目标组件、操作类型、方法参数、属性/样式 JSON |
| 数据 | `setVariable` | 变量路径、固定值或表达式 |
| 逻辑 | `condition` | 条件表达式、真假分支动作 |
| 高级 | `custom` | 自定义 JS 脚本，仅编辑器预览执行 |

组件联动类动作必须围绕目标组件配置：

- 显示/隐藏、启用/禁用：只需要目标组件和操作类型。
- 设置/清空值：需要值属性，并支持固定值、事件数据引用和表达式。
- 打开/关闭弹窗：目标组件只允许 Modal 或 Drawer。
- 提交/重置表单：目标组件只允许 Form。
- 调用组件方法：目标组件只展示 registry 声明了 methods 的组件。

## 事件数据约定

常见事件数据：

| 事件 | 典型数据 |
| --- | --- |
| `click` | 当前组件、原始事件参数 |
| `change` | `value`、当前组件、原始事件参数 |
| `finish` | `values` |
| `finishFailed` | `errorFields`、`values` |
| `valuesChange` | `changedValues`、`allValues` |
| `rowClick` | `record`、`index` |
| `loadSuccess` | `data` |
| `loadError` | `error` |

事件数据由 `src/editor/events/createEventData.ts` 生成，表达式和动作执行由 `src/editor/events/runtime.ts` 处理。

## 推荐优先级

### P0：稳定当前闭环

- 保持 Button、Text、Image、Card、Container、Alert、Input、Select、Switch、Form、Table 的常用事件可用。
- 保证动作弹窗编辑已有动作时不会丢配置。
- 保证 Preview 注入事件 props 后，prod 物料能正确转发。
- 保证公开页 `allowCustomJS=false`。

### P1：增强数据联动

- Table 远程数据加载事件。
- 表单提交结果写入变量。
- 组件属性绑定 `{{variables.xxx}}` 的体验优化。
- 数据源请求状态可视化。

### P2：高级编排

- 更完整的条件分支。
- 全局事件或页面生命周期。
- 动作执行日志和调试面板。
- 自定义 JS 沙箱化。

## 改事件系统时必须检查

- `src/editor/registry/component-config.tsx` 中是否声明事件。
- `src/editor/components/Setting/ComponentEvent.tsx` 是否能展示和编辑。
- `src/editor/components/Setting/ActionModal.tsx` 是否正确初始化已有动作。
- `src/editor/runtime/Preview/index.tsx` 是否注入对应 React/AntD 事件 prop。
- 对应 `materials/<Name>/prod.tsx` 是否转发事件 prop。
- `packages/lowcode-schema/src/migrate.ts` 是否需要兼容旧数据。
- `scripts/test/*.test.mjs` 是否需要补充覆盖。
