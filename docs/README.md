# 项目文档中心

本文档中心只保留对协作、开发、验收有长期价值的内容。临时计划、阶段复盘和学习材料不作为新任务的默认入口。

## 推荐入口

| 场景 | 先读 |
| --- | --- |
| AI 新对话快速接手 | `00-总览/AI快速上手.md` |
| 快速了解项目现状 | `00-总览/项目上下文索引.md` |
| 查关键文件和改动风险 | `.claude/context/FILE_MAP.md` |
| 理解架构和数据流 | `02-架构/架构说明.md` |
| 查接口、鉴权和 schema contract | `03-接口/接口说明.md` |
| 改编辑器事件能力 | `04-编辑器/事件绑定系统设计.md`、`04-编辑器/事件能力矩阵.md` |
| 本地运行和质量检查 | `05-开发/项目运行指南.md`、`05-开发/CI与部署基础.md` |
| 上线前验收 | `06-测试与验收/上线验收清单.md`、`07-安全/上线安全检查清单.md` |
| 多窗口并行协作 | `10-协作/并行任务看板.md` |

## 目录定位

| 目录 | 用途 | 维护原则 |
| --- | --- | --- |
| `00-总览` | AI 接手、项目概览、阅读路线 | 保持短，只写稳定事实和入口 |
| `01-产品` | 产品定位、需求、路线图 | 需求或优先级变化时更新 |
| `02-架构` | 前后端架构、数据流、技术决策 | 架构变化时同步 |
| `03-接口` | API、鉴权、错误格式、schema contract | 接口变化时同步 |
| `04-编辑器` | 物料、事件、编辑器体验 | 编辑器机制变化时同步 |
| `05-开发` | 运行、编码、Git、CI、部署、备份 | 命令和流程变化时同步 |
| `06-测试与验收` | 演示流程、上线验收 | 核心功能变化时同步 |
| `07-安全` | 鉴权、权限、发布页、自定义 JS、上线安全 | 安全边界变化时同步 |
| `08-复盘` | 里程碑摘要和验证记录 | 只保留阶段结论，不堆完整流水账 |
| `09-学习资料` | 学习、面试、讲解资料 | 不作为工程事实来源 |
| `10-协作` | 多窗口并行任务、资源锁和交接记录 | 开并行任务前更新 |
| `99-归档` | 历史方案 | 只读参考 |

## 精简原则

- 入口文档只回答“读什么、改哪里、怎么验”。
- 架构/API 文档只保留当前有效事实，细节以源码为准。
- 已完成的历史计划进入 `08-复盘` 或 `99-归档`，不要继续放在当前路线。
- 同一段流程只在一个地方详细描述，其他文档只链接或摘要。
- 新增功能后优先更新 `AI快速上手.md`、`项目上下文索引.md`、相关专题文档和 `FILE_MAP.md`。
