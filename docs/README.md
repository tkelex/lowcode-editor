# 项目文档中心

本文档中心按企业项目常见资料域整理，正式协作文档、开发规范、验收资料、复盘材料和学习材料分开存放。

## 推荐阅读

| 场景 | 推荐文档 |
| --- | --- |
| 快速了解项目 | `00-总览/项目上下文索引.md` |
| 理解产品目标 | `01-产品/需求文档.md`、`01-产品/后续路线图.md`、`01-产品/后续开发实施计划.md` |
| 理解技术架构 | `02-架构/架构说明.md`、`02-架构/技术决策记录.md` |
| 对接后端接口 | `03-接口/接口说明.md` |
| 继续完善编辑器事件、体验和物料 | `04-编辑器/事件能力矩阵.md`、`04-编辑器/事件绑定系统设计.md`、`04-编辑器/事件动作规划.md`、`04-编辑器/编辑器体验优化说明.md`、`04-编辑器/物料体系说明.md` |
| 本地运行和开发规范 | `05-开发/项目运行指南.md`、`05-开发/本地开发指南.md`、`05-开发/编码规范.md`、`05-开发/企业级工程规范化计划.md`、`05-开发/CI与部署基础.md` |
| 演示或验收项目 | `06-测试与验收/功能演示流程.md` |
| 查看安全要求 | `07-安全/安全规范.md` |
| 复盘历史进度 | `08-复盘/开发进度与学习总结.md` |
| 学习和面试准备 | `09-学习资料/后端与数据库说明.md`、`09-学习资料/面试讲解稿.md` |

## 目录说明

| 目录 | 内容定位 | 维护建议 |
| --- | --- | --- |
| `00-总览` | 项目入口、上下文索引、阅读顺序 | 保持简洁，只写稳定事实 |
| `01-产品` | 需求、路线图、产品规划 | 需求和优先级变化时更新 |
| `02-架构` | 架构说明、技术选型、技术决策 | 架构或关键技术选择变化时更新 |
| `03-接口` | API、鉴权、schema contract | 后端接口变化时同步 |
| `04-编辑器` | 编辑器专项设计，如事件动作、物料体系 | 编辑器机制变化时同步 |
| `05-开发` | 本地开发、编码规范、Git 流程 | 命令、流程、规范变化时同步 |
| `06-测试与验收` | 演示流程、验收清单、测试说明 | 新增核心功能后补充演示步骤 |
| `07-安全` | 安全规范、风险边界 | 鉴权、发布、自定义 JS、上传等能力变化时同步 |
| `08-复盘` | 开发进度、验证记录、学习总结 | 重要里程碑完成后追加 |
| `09-学习资料` | 给学习、面试、讲解使用的辅助材料 | 不作为正式需求或架构依据 |
| `99-归档` | 已过时或被新文档替代的历史资料 | 原则上只读，必要时再清理 |

## 清理原则

- 正式项目协作文档放在 `00-总览` 到 `07-安全`。
- 阶段复盘和学习记录放在 `08-复盘`。
- 面试讲解、概念解释这类资料放在 `09-学习资料`，不混入正式架构目录。
- 旧计划、被路线图替代的文档放入 `99-归档`。
- 不确定是否仍有价值的文档先归档，不直接删除。
