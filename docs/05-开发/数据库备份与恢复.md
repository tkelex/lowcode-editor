# 数据库备份与恢复

## 目标

上线前必须能备份和恢复 PostgreSQL。低代码项目的核心数据都在数据库里，包括用户、项目、页面 schema、版本、发布状态、素材记录和审计日志。

备份不等于完成任务，能恢复并验证数据才算真正可用。

## 备份方式

项目提供 PowerShell 脚本：

```powershell
scripts/db/backup-postgres.ps1
scripts/db/restore-postgres.ps1
```

### 本机 PostgreSQL 客户端方式

如果本机已经安装 `pg_dump`：

```powershell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/lowcode_editor?schema=public"
powershell -ExecutionPolicy Bypass -File scripts/db/backup-postgres.ps1
```

默认输出到：

```text
backups/lowcode-editor-YYYYMMDD-HHmmss.dump
```

### Docker Compose 方式

如果没有本机 PostgreSQL 客户端，但数据库跑在 `docker compose` 的 `postgres` 服务里：

```powershell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/lowcode_editor?schema=public"
powershell -ExecutionPolicy Bypass -File scripts/db/backup-postgres.ps1 -DockerComposeService postgres -ContainerDatabaseUrl "postgresql://postgres:postgres@localhost:5432/lowcode_editor?schema=public"
```

如果脚本在其它服务容器中执行，例如生产编排里的应用容器，数据库地址通常应使用服务名：

```text
postgresql://postgres:postgres@postgres:5432/lowcode_editor?schema=public
```

## 恢复方式

恢复到空数据库：

```powershell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/lowcode_editor?schema=public"
powershell -ExecutionPolicy Bypass -File scripts/db/restore-postgres.ps1 -BackupFile backups/lowcode-editor-20260509-120000.dump
```

恢复到已有数据库并清理旧对象：

```powershell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/lowcode_editor?schema=public"
powershell -ExecutionPolicy Bypass -File scripts/db/restore-postgres.ps1 -BackupFile backups/lowcode-editor-20260509-120000.dump -Clean
```

Docker Compose 方式：

```powershell
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/lowcode_editor?schema=public"
powershell -ExecutionPolicy Bypass -File scripts/db/restore-postgres.ps1 -BackupFile backups/lowcode-editor-20260509-120000.dump -DockerComposeService postgres -ContainerDatabaseUrl "postgresql://postgres:postgres@localhost:5432/lowcode_editor?schema=public" -Clean
```

## 上线前备份流程

每次上线前建议按这个顺序执行：

1. 停止会写入数据的后台任务。
2. 执行数据库备份。
3. 确认备份文件大小不是 0。
4. 执行 `npm.cmd run prisma:deploy --prefix server`。
5. 启动新版本后端。
6. 打开前端并完成上线验收清单。

## 恢复演练

上线前至少做一次恢复演练：

1. 新建一个测试数据库。
2. 把备份恢复到测试数据库。
3. 修改 `server/.env` 指向测试数据库。
4. 启动后端。
5. 检查：
   - 能登录。
   - 能看到项目。
   - 能打开页面。
   - 页面版本存在。
   - 已发布页面能访问。
   - 管理员后台能看到审计日志。

## 注意事项

- 不要把真实备份文件提交到 Git。
- 生产数据库账号不要使用超级用户。
- 备份文件可能包含用户邮箱、页面内容和审计信息，必须当作敏感数据保存。
- 本地 `backups/` 目录应加入忽略规则。
- 恢复前确认目标数据库，避免把生产库误清空。
