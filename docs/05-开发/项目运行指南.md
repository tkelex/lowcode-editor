# 项目运行指南

本文档只保留启动项目需要的关键命令。默认使用 Windows PowerShell，并在项目根目录执行：

```powershell
cd E:\前端\nk_React\project\lowcode-editor
```

## 首次运行

### 1. 安装依赖

```powershell
npm.cmd install
npm.cmd install --prefix server
```

### 2. 准备环境变量

```powershell
Copy-Item .env.example .env
Copy-Item server/.env.example server/.env
```

### 3. 启动数据库

```powershell
docker compose up -d postgres
docker compose ps
```

### 4. 初始化数据库

```powershell
npm.cmd run prisma:generate --prefix server
npm.cmd run prisma:migrate --prefix server
```

已有数据库后拉到新代码时，先应用已提交 migration，再重新生成 Prisma Client：

```powershell
npm.cmd run prisma:deploy --prefix server
npm.cmd run prisma:generate --prefix server
```

如果准备上线或升级真实数据，执行 migration 前先备份数据库，详见：

```text
docs/05-开发/数据库备份与恢复.md
```

## Docker 一键启动

完整本地栈包含 PostgreSQL、NestJS API、Nginx 前端：

```powershell
docker compose up --build
```

访问地址：

```text
前端：http://localhost:8080
后端：http://localhost:3000/api
健康检查：http://localhost:3000/api/health
```

停止完整栈：

```powershell
docker compose down
```

如需同时删除数据库和上传素材卷：

```powershell
docker compose down -v
```

## 管理员账号

普通注册用户默认不是平台管理员。需要先注册一个普通账号，再通过后端脚本授予管理员角色：

```powershell
npm.cmd run admin:grant --prefix server -- --email admin@example.com
```

授予成功后重新登录该账号，项目面板右上角会出现“管理后台”入口，也可以直接访问：

```text
http://localhost:5173/admin
```

生产环境不要开放“注册管理员”接口，管理员账号应通过脚本或内部运维流程授予。

## 常用检查

```powershell
npm.cmd run build
npm.cmd run build --prefix server
npm.cmd run check
npm.cmd run test:e2e
npm.cmd run test:e2e:editor
npm.cmd run smoke:api
```

`test:e2e` 会自动启动前端开发服务器，并使用 mock API 验证前端主链路。首次运行如缺少浏览器，可执行：

```powershell
npx.cmd playwright install chromium
```

`smoke:api` 需要先启动数据库和后端，用来验证真实后端接口链路。

`test:e2e:editor` 是编辑器复盘用例，重点检查右侧设置面板、横向滚动、预览模式和输入类组件渲染。后续改编辑器 UI 或事件配置时，优先跑这一条。

如果你准备正式上线，建议直接看：

```text
docs/05-开发/部署与运维指南.md
```
