# 开发进度与学习总结

本文只保留阶段性结论、已验证能力和后续学习重点。详细实现以源码、提交记录和专题文档为准。

## 当前阶段结论

项目已经从前端 demo 演进为具备核心闭环的全栈低代码编辑器：

- 前端：React 编辑器、物料区、画布、设置面板、预览、源码面板、响应式画布、大纲树、右键菜单、撤销重做。
- 后端：NestJS API、JWT 鉴权、项目/页面/成员/版本/发布/审计。
- 数据库：PostgreSQL + Prisma，保存用户、项目、页面、版本、审计等数据。
- 共享能力：`packages/lowcode-schema` 统一 schema 类型、迁移、校验和父子规则。
- 工程化：lint、build、test、API smoke、e2e、CI、Docker/Nginx、部署和备份文档。

## 已完成的主要能力

### 用户和项目

- 注册、登录、读取当前用户。
- 创建、读取、更新、删除项目。
- 项目成员 owner/editor/viewer。
- 成员添加、角色变更、移除。
- 项目和页面关键操作审计。

### 页面和编辑器

- 创建页面、打开页面、保存页面 schema。
- 组件树持久化到 PostgreSQL。
- 本地缓存恢复时执行 schema migration。
- 源码面板可编辑、校验并应用 JSON。
- 版本历史、回滚、删除版本记录。
- 保存、发布、回滚都经过 schema 迁移和校验。

### 发布和运行态

- 页面发布生成公开 `publicId`。
- 公开页读取 `publishedVersionId` 指向的发布快照。
- 发布页渲染使用 Preview，且禁用 customJS。
- 编辑器预览支持 toast、url、componentAction、confirm、condition、http、setVariable、setComponentProps、setComponentStyles、custom 等动作。

### 管理和上线

- 平台管理员角色和账号状态。
- 管理员 API 和前端 `/admin` 页面。
- 用户禁用、项目禁用、发布页取消发布、全局审计。
- 生产环境变量校验、部署模板、备份恢复、上线验收和安全检查文档。

## 已验证内容

最近记录中已通过或已加入验证链路的内容：

- `npm run build`
- `npm run build --prefix server`
- `npm run test`
- `npm run check`
- `npm run smoke:api`
- schema migration、schema 校验、URL normalize。
- 注册、登录、成员权限、页面保存、版本、发布、公开读取、取消发布、审计的 API smoke。

具体是否在当前工作区仍通过，需要按任务重新运行命令确认。

## 当前重点风险

- 工作区可能存在大量未提交改动，新任务开始前必须看 `git status --short`。
- 组件树 store、物料 registry、Preview 运行态和共享 schema 是高风险区域。
- 文档可能滞后于源码，最终判断以源码和自动化验证为准。
- 公开发布页不能执行用户自定义 JS。
- 权限必须由后端强制校验。

## 后续推荐顺序

1. 先做一次项目体检：lint、build、test、e2e、关键 UI 截图。
2. 整理 bug 清单，按 P0/P1/P2 修复。
3. 优化编辑器样式和交互，重点看三栏布局、设置面板、画布、弹窗。
4. 补成员管理 UI、审计 UI、版本差异对比。
5. 完善模板、区块、素材管理。
6. 推进数据源、变量、表达式联动。
7. 做真实部署演练和上线验收。

## 学习重点

如果目的是系统掌握项目，推荐按这个顺序学：

1. 先跑通本地项目和常用命令。
2. 理解前端应用入口和视图切换。
3. 理解组件树 store、物料 registry、dev/prod 渲染。
4. 理解事件系统和 Preview 运行态。
5. 理解后端 API、JWT、项目权限。
6. 理解 Prisma 模型、schema 保存、版本和发布。
7. 理解测试、CI、部署、安全和备份。

详细学习路径见 `docs/09-学习资料/项目系统学习计划.md`。
