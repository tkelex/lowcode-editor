# API 说明

本文记录当前 API 分组、鉴权规则、关键请求和 schema contract。字段细节以 controller、dto、service 和 `src/shared/api/*` 为准。

## 基础信息

```text
本地后端：http://localhost:3000/api
前端配置：VITE_API_BASE_URL=http://localhost:3000/api
Token key：lowcode_editor_token
鉴权头：Authorization: Bearer <accessToken>
```

统一错误响应包含：

```json
{
  "statusCode": 400,
  "code": "PAGE_SCHEMA_INVALID",
  "message": "错误信息",
  "path": "/api/pages/1",
  "method": "PATCH",
  "requestId": "request-id",
  "timestamp": "2026-04-28T00:00:00.000Z"
}
```

## 鉴权边界

公开接口：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/public/pages/:publicId`

受保护接口：

- 项目、页面、版本、发布、素材、模板等业务接口需要 JWT。
- 管理员接口需要 JWT + `AdminGuard`。

项目角色：

| 角色 | 能力 |
| --- | --- |
| owner | 管理项目、成员、审计、页面、版本、发布 |
| editor | 读项目，创建/保存/发布/回滚页面 |
| viewer | 只读项目、页面、版本 |

## Auth API

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/auth/register` | 注册并返回 token |
| POST | `/api/auth/login` | 登录并返回 token |
| GET | `/api/auth/me` | 获取当前用户 |

注册：

```json
{
  "email": "user@example.com",
  "username": "username",
  "password": "password123"
}
```

登录：

```json
{
  "account": "username or email",
  "password": "password123"
}
```

响应核心结构：

```json
{
  "accessToken": "jwt-token",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "username": "username",
    "role": "user",
    "status": "active"
  }
}
```

## Projects API

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| GET | `/api/projects` | 项目成员 | 当前用户可访问项目 |
| POST | `/api/projects` | 登录用户 | 创建项目，自动成为 owner |
| GET | `/api/projects/:id` | 项目成员 | 项目详情 |
| PATCH | `/api/projects/:id` | owner | 更新项目 |
| DELETE | `/api/projects/:id` | owner | 删除项目 |
| GET | `/api/projects/:id/members` | 项目成员 | 成员列表 |
| POST | `/api/projects/:id/members` | owner | 添加成员 |
| PATCH | `/api/projects/:id/members/:memberId` | owner | 修改成员角色 |
| DELETE | `/api/projects/:id/members/:memberId` | owner | 移除成员 |
| GET | `/api/projects/:id/audit-logs` | owner | 项目审计 |

成员角色只支持新增或调整为 `editor/viewer`，不能通过成员接口新增或移除 owner。

创建页面：

```json
{
  "name": "首页",
  "routePath": "/home"
}
```

```json
{
  "name": "首页",
  "routePath": "/home",
  "schema": {
    "schemaVersion": "1.0.0",
    "pageId": 1,
    "components": [
      {
        "id": 1,
        "name": "Page",
        "props": {},
        "desc": "页面",

      }
    ],
    "metadata": {
      "updatedAt": "2026-04-28T00:00:00.000Z"
    }
```

## AI Page Builder API

AI 页面搭建接口需要 JWT，并且只允许项目 `owner/editor` 使用。接口只返回低代码 Page schema 草稿，不生成 React/Vue/HTML 源码，也不会把模型 API key 暴露给前端。服务端未配置 `AI_PROVIDER_API_KEY` 时，会返回本地规则生成的可编辑草稿，并在 `warnings` 中说明降级原因。

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| POST | `/api/projects/:projectId/ai/page-generation` | owner/editor | 基于项目上下文生成页面草稿 |
| POST | `/api/pages/:pageId/ai/page-generation` | owner/editor | 基于当前页面上下文生成页面草稿 |
| POST | `/api/projects/:projectId/ai/agent-runs` | owner/editor | 基于项目上下文创建 AI agent run |
| POST | `/api/pages/:pageId/ai/agent-runs` | owner/editor | 基于当前页面和组件树创建 AI agent run |
| GET | `/api/ai/agent-runs/:runId` | 项目成员 | 读取 agent run 状态和候选结果 |
| POST | `/api/ai/agent-runs/:runId/cancel` | 项目成员 | 取消仍在执行的 agent run |

请求：

```json
{
  "prompt": "生成一个用户管理页面，包含统计、列表和编辑表单",
  "target": "crud",
  "writeMode": "replacePage",
  "apiDescription": "GET /api/users 返回用户列表",
  "responseSample": {
    "data": {
      "items": [
        { "id": 1, "name": "张三", "status": "启用" }
      ]
    }
  }
}
```

响应：

```json
{
  "summary": "已生成用户管理页面草稿",
  "warnings": ["CRUD 生成器对接点已预留"],
  "assumptions": ["字段类型根据接口示例自动推断"],
  "components": [
    {
      "id": 1,
      "name": "Page",
      "props": {},
      "desc": "页面",
      "children": []
    }
  ],
  "metadata": {
    "source": "model"
  }
}
```

Agent run 请求：

```json
{
  "prompt": "给当前容器增加一个搜索区",
  "targetScope": "selection",
  "selectedComponentId": 12,
  "currentComponents": [],
  "apiDescription": "GET /api/users 返回用户列表",
  "responseSample": {
    "data": {
      "items": [
        { "id": 1, "name": "张三" }
      ]
    }
  }
}
```

Agent run 响应核心结构：

```json
{
  "runId": "agent_...",
  "status": "awaiting_confirmation",
  "plan": ["读取当前页面上下文", "生成低代码 schema 草稿", "校验候选修改"],
  "events": [
    { "type": "tool_call", "title": "已读取页面上下文" }
  ],
  "candidate": {
    "kind": "patch",
    "summary": "已生成候选修改",
    "impactScope": "selection",
    "patch": {
      "operations": [
        { "type": "addChild", "parentId": 12, "component": { "id": 1, "name": "Text", "props": {}, "desc": "文本" } }
      ]
    },
    "previewComponents": []
  }
}
```

安全规则：

- AI 输出写入编辑器前必须通过物料白名单、组件树、父子关系和事件动作校验。
- AI 默认不得生成 `custom` action 或可执行脚本。
- AI agent 只能调用服务端白名单工具；工具只返回候选 schema、候选 patch 或校验结果，不直接保存、发布或访问任意外部系统。
- AI agent 候选 patch 必须由用户确认后才能写入编辑器，应用时会重新校验 stale baseline、目标组件、父子关系和安全规则。
- 公开发布页继续通过 `<Preview allowCustomJS={false} />` 禁止自定义 JS。
- HTTP action 仍受现有 URL、headers、body 和 allowed origins 运行态规则约束。

| 方法 | 路径 | 说明 |
| --- | --- | --- |

## Schema Contract

组件节点：

```ts
{
  id: number;
  name: string;
  props: any;
  styles?: CSSProperties;
  desc: string;
  children?: Component[];
  parentId?: number;
}
```

页面 schema：

```ts
{
  schemaVersion: CURRENT_SCHEMA_VERSION;
  pageId?: number;
  components: Component[];
  metadata?: Record<string, unknown>;
}
```

关键规则：

- 前端保存前校验 `components`。
- 后端创建、保存、发布、回滚前都会先迁移再校验。
- AI 生成结果写入编辑器前会先通过 `validateAiGeneratedComponents` 校验，再由用户确认应用。
- `Page.schema` 是当前草稿。
- `PageVersion.schema` 是保存、回滚或发布时的快照。
- 公开页只读取 `publishedVersionId` 指向的版本快照。
