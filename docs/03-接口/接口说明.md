# API 说明

## 基础信息

后端框架：NestJS

默认 API 前缀：

```text
/api
```

默认本地地址：

```text
http://localhost:3000/api
```

前端通过 `VITE_API_BASE_URL` 配置 API 地址：

```text
VITE_API_BASE_URL=http://localhost:3000/api
```

## 鉴权方式

登录成功后后端返回 access token。

前端保存到 localStorage key：

```text
lowcode_editor_token
```

请求业务接口时通过 header 传递：

```http
Authorization: Bearer <accessToken>
```

前端封装位置：`src/shared/api/http.ts`

后端鉴权相关文件：

```text
server/src/common/guards/jwt-auth.guard.ts
server/src/common/guards/jwt.strategy.ts
server/src/common/decorators/current-user.decorator.ts
```

## 统一错误格式

后端使用全局异常过滤器：

```text
server/src/common/filters/http-exception.filter.ts
```

错误响应大致形态：

```json
{
  "statusCode": 400,
  "code": "BAD_REQUEST",
  "message": "错误信息",
  "path": "/api/example",
  "method": "POST",
  "timestamp": "2026-04-28T00:00:00.000Z"
}
```

常见业务错误码：

```text
AUTH_ACCOUNT_EXISTS
AUTH_INVALID_CREDENTIALS
AUTH_USER_NOT_FOUND
PROJECT_NOT_FOUND
PROJECT_FORBIDDEN
PROJECT_MEMBER_NOT_FOUND
PROJECT_MEMBER_CONFLICT
PROJECT_OWNER_IMMUTABLE
PAGE_NOT_FOUND
PAGE_ROUTE_CONFLICT
PAGE_SCHEMA_INVALID
PAGE_VERSION_NOT_FOUND
PUBLISHED_PAGE_NOT_FOUND
CONFIG_INVALID
```

## Auth API

Controller：`server/src/modules/auth/auth.controller.ts`

Service：`server/src/modules/auth/auth.service.ts`

### POST /api/auth/register

注册用户。

请求体：

```json
{
  "email": "user@example.com",
  "username": "username",
  "password": "password123"
}
```

校验：

- `email` 必须是邮箱。
- `username` 长度 3-32。
- `password` 长度 6-64。

响应：

```json
{
  "accessToken": "jwt-token",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "username": "username",
    "nickname": null,
    "avatarUrl": null
  }
}
```

### POST /api/auth/login

登录用户。

请求体：

```json
{
  "account": "username 或 email",
  "password": "password123"
}
```

响应同注册。

### GET /api/auth/me

获取当前登录用户。

需要 JWT。

响应：

```json
{
  "id": 1,
  "email": "user@example.com",
  "username": "username",
  "nickname": null,
  "avatarUrl": null
}
```

## Projects API

Controller：`server/src/modules/projects/projects.controller.ts`

Service：`server/src/modules/projects/projects.service.ts`

所有接口都需要 JWT。

项目角色：

```text
owner   项目拥有者：可管理项目、成员、审计和页面
editor  项目编辑者：可读取项目、读写页面、发布/回滚页面
viewer  项目查看者：只能读取项目、成员、页面和版本
```

### GET /api/projects

获取当前用户拥有或被邀请加入的项目列表。

响应会包含当前用户在项目里的角色：

```json
[
  {
    "id": 1,
    "ownerId": 1,
    "name": "项目名称",
    "description": "项目描述",
    "status": "active",
    "currentUserRole": "owner",
    "createdAt": "2026-04-28T00:00:00.000Z",
    "updatedAt": "2026-04-28T00:00:00.000Z"
  }
]
```

### POST /api/projects

创建项目。

请求体：

```json
{
  "name": "项目名称",
  "description": "项目描述"
}
```

响应示例：

```json
{
  "id": 1,
  "ownerId": 1,
  "name": "项目名称",
  "description": "项目描述",
  "status": "active",
  "currentUserRole": "owner",
  "createdAt": "2026-04-28T00:00:00.000Z",
  "updatedAt": "2026-04-28T00:00:00.000Z"
}
```

### GET /api/projects/:id

获取项目详情。

权限：项目成员可读取。响应包含 `currentUserRole`。

### PATCH /api/projects/:id

更新项目。

请求体：

```json
{
  "name": "新项目名称",
  "description": "新描述"
}
```

权限：仅 owner 可更新项目基础信息。

### DELETE /api/projects/:id

删除项目。

权限：仅 owner 可删除项目。

删除项目会 cascade 删除其页面。

### GET /api/projects/:id/members

获取项目成员列表。

权限：项目成员可读取。

响应示例：

```json
[
  {
    "id": 1,
    "projectId": 1,
    "userId": 1,
    "role": "owner",
    "user": {
      "id": 1,
      "email": "owner@example.com",
      "username": "owner",
      "nickname": null,
      "avatarUrl": null
    },
    "createdAt": "2026-04-28T00:00:00.000Z",
    "updatedAt": "2026-04-28T00:00:00.000Z"
  }
]
```

### POST /api/projects/:id/members

新增项目成员。

权限：仅 owner 可操作。

请求体可以用邮箱或用户 ID 指定已有用户：

```json
{
  "email": "editor@example.com",
  "role": "editor"
}
```

或：

```json
{
  "userId": 2,
  "role": "viewer"
}
```

当前接口不支持直接新增另一个 owner；owner 转移后续单独设计，避免项目没有明确拥有者。

### PATCH /api/projects/:id/members/:memberId

变更成员角色。

权限：仅 owner 可操作。

请求体：

```json
{
  "role": "viewer"
}
```

只能在 `editor/viewer` 之间变更，不能通过该接口修改项目 owner。

### DELETE /api/projects/:id/members/:memberId

移除项目成员。

权限：仅 owner 可操作。不能移除项目 owner。

响应：

```json
{
  "success": true
}
```

### GET /api/projects/:id/audit-logs

获取项目审计日志，默认返回最近 200 条。

权限：仅 owner 可读取。

响应示例：

```json
[
  {
    "id": 1,
    "actorId": 1,
    "projectId": 1,
    "pageId": 2,
    "action": "page.publish",
    "targetType": "page",
    "targetId": "2",
    "summary": "Publish page 首页",
    "metadata": {
      "publicId": "ckxxxx",
      "publishedVersionId": 3
    },
    "createdAt": "2026-04-28T00:00:00.000Z"
  }
]
```

## Pages API

Controller：`server/src/modules/pages/pages.controller.ts`

Service：`server/src/modules/pages/pages.service.ts`

受保护页面接口需要 JWT；公开发布页读取接口不需要 JWT。

页面权限规则：

- `owner/editor` 可以创建、保存、发布、取消发布、回滚、删除页面和删除历史版本。
- `viewer` 只能读取页面列表、页面详情和历史版本。
- 非项目成员访问页面会被拒绝。

### GET /api/projects/:projectId/pages

获取某个项目下的页面列表。

权限：项目成员可读取。

### POST /api/projects/:projectId/pages

创建页面。

权限：owner/editor 可创建。

请求体：

```json
{
  "name": "首页",
  "routePath": "/home"
}
```

也可以传入初始 schema：

```json
{
  "name": "首页",
  "routePath": "/home",
  "schema": {
    "schemaVersion": "1.0.0",
    "components": [
      {
        "id": 1,
        "name": "Page",
        "props": {},
        "desc": "页面"
      }
    ],
    "metadata": {}
  }
}
```

如果不传 schema，后端会写入默认 Page schema。

后端会在入库前通过 `packages/lowcode-schema/src/migrate.ts` 迁移 schema，再通过共享校验器校验组件树。旧事件字段会被收敛到 `props.onEvent`，缺省 schema 会补为默认 Page 根节点。

`routePath` 当前格式：

```text
必须以 / 开头，只允许字母、数字、下划线、中划线和斜杠
```

### GET /api/pages/:id

获取页面详情。

权限：项目成员可读取。

响应核心字段：

```json
{
  "id": 1,
  "projectId": 1,
  "createdById": 1,
  "name": "首页",
  "routePath": "/home",
  "schema": {
    "schemaVersion": "1.0.0",
    "pageId": 1,
    "components": [],
    "metadata": {}
  },
  "publicId": "ckxxxx-public-id",
  "isPublished": false,
  "publishedAt": null,
  "publishedVersionId": null,
  "createdAt": "2026-04-28T00:00:00.000Z",
  "updatedAt": "2026-04-28T00:00:00.000Z"
}
```

### PATCH /api/pages/:id

更新页面或保存 schema。

权限：owner/editor 可更新。

请求体：

```json
{
  "name": "首页",
  "routePath": "/home",
  "schema": {
    "schemaVersion": "1.0.0",
    "pageId": 1,
    "components": [
      {
        "id": 1,
        "name": "Page",
        "props": {},
        "desc": "页面",
        "children": []
      }
    ],
    "metadata": {
      "updatedAt": "2026-04-28T00:00:00.000Z"
    }
  }
}
```

前端保存按钮调用位置：`src/editor/components/Header/index.tsx`

前端 API 封装位置：`src/shared/api/pages.ts`

### GET /api/pages/:id/versions

获取页面历史版本列表。

权限：项目成员可读取。

响应示例：

```json
[
  {
    "id": 1,
    "pageId": 1,
    "createdById": 1,
    "versionNo": 1,
    "schema": {
      "schemaVersion": "1.0.0",
      "pageId": 1,
      "components": [],
      "metadata": {}
    },
    "source": "save",
    "message": null,
    "createdAt": "2026-04-28T00:00:00.000Z"
  }
]
```

`source` 当前含义：

```text
save      用户点击保存生成的版本
rollback  用户执行回滚后生成的新版本
publish   用户发布页面时生成的公开访问快照
```

### POST /api/pages/:id/rollback

回滚页面到指定历史版本。

权限：owner/editor 可回滚，并且版本必须属于该页面。

请求体：

```json
{
  "versionId": 1
}
```

响应：更新后的页面。

回滚行为会：

1. 把目标 `PageVersion.schema` 恢复到 `Page.schema`。
2. 新建一条 `source = "rollback"` 的 PageVersion，记录本次回滚结果。

### DELETE /api/pages/:id/versions/:versionId

删除某条页面历史版本记录。

权限：owner/editor 可删除，并且版本必须属于该页面。

响应：

```json
{
  "success": true
}
```

删除行为只会删除 `PageVersion` 记录，不会修改当前 `Page.schema`，也不会影响编辑器当前页面内容。删除后的历史版本不能再用于回滚。

### POST /api/pages/:id/publish

发布当前页面。

权限：owner/editor 可发布。

行为：

1. 基于当前数据库中的 `Page.schema` 创建一条 `source = "publish"` 的 `PageVersion`。
2. 如果旧页面还没有 `publicId`，发布时生成一个不可枚举的公开 ID。
3. 更新 `Page.isPublished = true`、`Page.publishedAt` 和 `Page.publishedVersionId`。
4. 返回更新后的页面。

响应核心字段：

```json
{
  "id": 1,
  "publicId": "ckxxxx-public-id",
  "isPublished": true,
  "publishedAt": "2026-04-28T00:00:00.000Z",
  "publishedVersionId": 3,
  "schema": {
    "schemaVersion": "1.0.0",
    "pageId": 1,
    "components": [],
    "metadata": {}
  }
}
```

前端发布成功后会生成公开地址：

```text
/publish/:publicId
```

前端发布按钮会先把当前画布组件树保存到 `Page.schema`，再调用发布接口生成发布快照。后端发布接口本身仍然只基于数据库中的 `Page.schema` 创建快照。

### POST /api/pages/:id/unpublish

取消发布当前页面。

权限：owner/editor 可取消发布。

行为：

- 设置 `Page.isPublished = false`。
- 保留 `publishedVersionId` 和历史 `PageVersion` 快照。
- 取消发布后，公开接口会返回 404。

### GET /api/public/pages/:publicId

公开读取已发布页面。

不需要 JWT。

行为：

1. 根据 `publicId` 查找 `isPublished = true` 的页面。
2. 读取 `Page.publishedVersionId` 指向的 `PageVersion.schema`。
3. 只返回公开页面需要的字段。

响应：

```json
{
  "publicId": "ckxxxx-public-id",
  "name": "首页",
  "routePath": "/home",
  "schema": {
    "schemaVersion": "1.0.0",
    "pageId": 1,
    "components": [],
    "metadata": {}
  },
  "publishedAt": "2026-04-28T00:00:00.000Z"
}
```

公开接口不会返回 `createdById`、`projectId`、`createdBy`、`project` 等内部信息。

### DELETE /api/pages/:id

删除页面。

权限：owner/editor 可删除页面。

## Schema contract

前端编辑器当前组件树来自 Zustand：`src/editor/stores/components.tsx`

单个组件节点：

```ts
{
  id: number;
  name: string;
  props: any;
  styles?: CSSProperties;
  desc: string;
  children?: Component[];
  parentId?: number;
}
```

保存到后端时包装为：

```ts
{
  schemaVersion: CURRENT_SCHEMA_VERSION,
  pageId,
  components,
  metadata: {
    updatedAt: new Date().toISOString()
  }
}
```

数据库字段：

```prisma
ProjectMember.projectId Int
ProjectMember.userId Int
ProjectMember.role ProjectMemberRole
Page.schema Json
Page.publicId String?
Page.isPublished Boolean
Page.publishedAt DateTime?
Page.publishedVersionId Int?
PageVersion.schema Json
AuditLog.actorId Int?
AuditLog.projectId Int?
AuditLog.pageId Int?
AuditLog.action String
AuditLog.metadata Json?
```

`Page.schema` 是当前编辑草稿，`PageVersion.schema` 是保存、回滚或发布时生成的快照。公开发布页读取 `publishedVersionId` 指向的版本快照，不直接读取可变的 `Page.schema`。

PostgreSQL 中 JSON 字段实际是 JSONB。

当前 schema 版本常量：

```text
packages/lowcode-schema/src/defaults.ts
CURRENT_SCHEMA_VERSION = 1.0.0
```

前端在打开页面、版本回滚、公开发布页渲染和本地缓存恢复时会先通过共享迁移器规范化 `schema`。源码应用、保存和发布前会通过共享校验器校验 `schema.components`，避免未注册物料、重复组件 id、非法 props/styles、错误父子关系等问题进入保存和发布流程。

共享 schema 能力位置：

```text
packages/lowcode-schema/src/defaults.ts
packages/lowcode-schema/src/migrate.ts
packages/lowcode-schema/src/validate.ts
packages/lowcode-schema/src/registry.ts
```

迁移器会补齐缺省字段、生成默认 Page schema、稳定分配异常组件 id，并把旧事件结构迁移到 `props.onEvent[eventName].actions`。后端 `server/src/modules/pages/page-schema.service.ts` 同样使用这套迁移和校验能力。保存、发布或回滚遇到非法 schema 时返回：

```json
{
  "statusCode": 400,
  "code": "PAGE_SCHEMA_INVALID",
  "message": "组件 schema 不合法",
  "path": "/api/pages/1",
  "method": "PATCH",
  "timestamp": "2026-04-28T00:00:00.000Z"
}
```

## API smoke test 路径

脚本位置：

```text
scripts/smoke/api-smoke.mjs
```

本地运行：

```bash
npm.cmd run smoke:api
```

最小验证顺序：

```text
POST /api/auth/register 注册 owner/editor/viewer 三个用户
GET /api/auth/me
POST /api/projects
GET /api/projects/:projectId/members 确认 owner 成员自动创建
GET /api/projects/:projectId/pages 非成员访问应返回 403
POST /api/projects/:projectId/members 添加 editor/viewer
POST /api/projects/:projectId/pages viewer 创建页面应返回 403
POST /api/projects/:projectId/pages editor 创建页面
PATCH /api/pages/:id viewer 保存页面应返回 403
PATCH /api/pages/:id editor 保存页面
GET /api/pages/:id/versions viewer 可读取版本
POST /api/pages/:id/publish editor 可发布
GET /api/public/pages/:publicId 不带 JWT 读取公开快照
POST /api/pages/:id/unpublish editor 可取消发布
PATCH /api/projects/:projectId/members/:memberId owner 将 editor 降级为 viewer
POST /api/pages/:id/publish 降级后的 editor 应返回 403
DELETE /api/projects/:projectId/members/:memberId owner 移除 viewer
GET /api/projects/:projectId/audit-logs owner 可读取审计日志
GET /api/projects/:projectId/audit-logs 非 owner 应返回 403
```

确认最后读取出的 `schema.components` 与保存内容一致，并且权限边界符合 owner/editor/viewer 规则。

版本回滚、删除和发布验证顺序：

```text
PATCH /api/pages/:id 保存版本一
PATCH /api/pages/:id 保存版本二
GET /api/pages/:id/versions
POST /api/pages/:id/rollback 回滚到版本一
GET /api/pages/:id 确认 schema 恢复
GET /api/pages/:id/versions 确认新增 rollback 版本
DELETE /api/pages/:id/versions/:versionId 删除某条历史版本
GET /api/pages/:id/versions 确认该版本消失
GET /api/pages/:id 确认当前 schema 不受删除影响
POST /api/pages/:id/publish 发布当前数据库中的 Page.schema
GET /api/public/pages/:publicId 不带 JWT 读取公开快照
PATCH /api/pages/:id 保存草稿但不重新发布
GET /api/public/pages/:publicId 确认公开页面仍是旧发布快照
POST /api/pages/:id/publish 再次发布
GET /api/public/pages/:publicId 确认公开页面更新
POST /api/pages/:id/unpublish 取消发布
GET /api/public/pages/:publicId 确认返回 404
```
