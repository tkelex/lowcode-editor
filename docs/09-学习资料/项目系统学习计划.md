# 项目系统学习计划

这份计划用于从“能跑起来”逐步过渡到“能维护、能排查、能讲清楚”。工程事实以源码和正式文档为准。

## 学习目标

- 能讲清楚项目做什么、为什么这样设计。
- 能定位前端、后端、数据库、权限、发布相关问题。
- 能完成小范围功能修改并运行对应验证。
- 能在面试或复盘中解释核心技术取舍。

## 学习原则

- 先跑通，再读源码。
- 先主链路，再细节。
- 每学一块都对应到文件。
- 每改一处都跑相应验证。

## 推荐阶段

| 阶段 | 目标 | 必读 |
| --- | --- | --- |
| 0. 全局地图 | 知道项目有哪些模块 | `docs/README.md`、`AI快速上手.md`、`FILE_MAP.md` |
| 1. 本地运行 | 能启动前端、后端、数据库 | `项目运行指南.md`、`CI与部署基础.md` |
| 2. 应用入口 | 理解登录、项目面板、编辑器、公开页切换 | `src/app/App.tsx`、`AppViewOutlet`、`useAuthSession` |
| 3. 编辑器核心 | 理解组件树、物料、画布、设置面板 | `src/editor/index.tsx`、`stores/components.tsx`、`component-config.tsx` |
| 4. 事件系统 | 理解事件配置和动作执行 | `src/editor/events/*`、`runtime/Preview` |
| 5. 后端 API | 理解鉴权、项目、页面和权限 | `server/src/modules/*`、`docs/03-接口/接口说明.md` |
| 6. 数据库 | 理解 Prisma 模型和 migration | `server/prisma/schema.prisma` |
| 7. 发布和安全 | 理解发布快照、公开页、customJS 边界 | `PublishedPageView`、`page-publish.service.ts` |
| 8. 测试部署 | 理解 check、smoke、e2e、CI、部署 | `scripts/`、`.github/`、`infra/` |

## 4 周节奏

### 第 1 周：跑起来和看懂前端壳

- 启动数据库、后端、前端。
- 注册用户，创建项目和页面。
- 读 `src/app` 和 `src/shared/api`。
- 跑 `npm run check`。

### 第 2 周：编辑器和事件

- 读组件树 store。
- 读物料 registry 和几个 material。
- 读事件配置和 Preview 运行态。
- 做一个小修复，例如修物料默认属性或样式问题。

### 第 3 周：后端、权限和数据库

- 读 Prisma schema。
- 读 auth、projects、pages 模块。
- 跑 API smoke。
- 理解 owner/editor/viewer 边界。

### 第 4 周：发布、测试和表达

- 读发布和公开页链路。
- 跑 e2e 编辑器回归。
- 按 `功能演示流程.md` 演示项目。
- 用 `面试讲解稿.md` 练习表达。

## 学到什么程度算达标

- 能说清楚组件树如何从后端进入编辑器。
- 能说清楚保存、版本、发布三者的区别。
- 能说清楚公开页为什么不能执行 customJS。
- 能定位一个物料在 dev、prod、registry、schema registry 中的对应位置。
- 能根据任务选择 `lint/build/test/smoke/e2e` 中的验证命令。
