# 项目后续建设计划

## 目标

把当前低代码编辑器继续推进到企业级低代码平台雏形：不仅能搭页面，还要能协作、能管理数据源、能发布、能审计、能部署、能持续迭代。

当前项目已经具备编辑器核心闭环、事件动作、P3 物料、后端保存发布、项目成员权限和基础审计。后续重点不再是单点补功能，而是围绕“企业实际使用”补齐协作、数据、资产、发布和工程治理。

## 当前 Git 状态建议

当前工作区包含多阶段大批量未提交改动，不建议直接一次性提交。建议按下面顺序拆分提交：

1. 文档整理与项目上下文更新。
2. 前端编辑器体验、事件动作、P3 物料和 schema 共享包。
3. 后端保存发布、版本、权限成员、审计和 API smoke。
4. CI、E2E、Docker、部署基础设施。

每次提交前至少运行：

```bash
npm.cmd run check
```

涉及后端接口、权限、数据库时额外运行：

```bash
npm.cmd run smoke:api
```

## P4 剩余：协作与企业化能力

### P4.1 成员管理前端 UI

目标：让后端已有的 `owner/editor/viewer` 能力变成用户可操作的界面。

任务：

- 在项目面板增加成员管理入口。
- 展示成员列表、角色、邮箱、用户名、加入时间。
- owner 可添加成员、修改 editor/viewer、移除成员。
- viewer/editor 看得到成员列表，但没有管理按钮。
- 根据 `currentUserRole` 控制新建页面、保存、发布、删除等按钮状态。

验收：

- viewer 进入项目后只能浏览，不能新建/保存/发布。
- editor 可编辑页面，但不能管理成员和项目设置。
- owner 可以完整管理项目。

### P4.2 审计日志前端 UI

目标：让 owner 能看到项目关键操作记录。

任务：

- 新增项目审计日志视图。
- 展示操作人、动作、对象、摘要、时间。
- 对 `page.publish`、`page.rollback`、`project.member.add` 等动作做中文标签。
- 后端支持按 action、pageId、时间范围筛选。

验收：

- 保存、发布、回滚、删除、成员变更都能在审计列表看到。
- 非 owner 访问审计接口会被拒绝，前端也不展示入口。

### P4.3 页面版本差异对比

目标：版本历史不只是回滚，还能看出改了什么。

任务：

- 后端提供版本详情接口。
- 前端版本历史弹窗增加“对比当前版本”和“对比两个版本”。
- 先做 JSON diff，再逐步升级为组件树语义 diff。
- 标识新增组件、删除组件、属性变更、样式变更、事件变更。

验收：

- 用户能判断一次保存主要改了哪些组件。
- 回滚前能先查看目标版本和当前草稿的差异。

### P4.4 模板和区块后端存储

目标：让页面模板、常用区块不只写死在前端代码里。

任务：

- 新增 `Template` 或 `PageTemplate` 数据模型。
- 支持页面模板和区块模板两种类型。
- 模板保存 schema、封面、分类、标签、创建人、可见性。
- 物料区模板从后端读取，保留前端内置模板作为兜底。
- 支持从当前页面或选中组件保存为模板。

验收：

- 用户可以把常用表单、表格、详情区块保存成模板。
- 新建页面时可以选择模板生成初始页面。

### P4.5 素材管理

目标：图片和文件不再只能手填 URL。

任务：

- 新增 `Asset` 数据模型。
- 后端上传接口，先支持本地磁盘存储，后续可切换对象存储。
- 支持图片、普通文件的大小限制、类型白名单、访问 URL。
- 图片组件、Upload 物料接入素材选择。
- 审计上传、删除素材操作。

验收：

- 用户可上传图片并在 Image 物料中选择使用。
- 发布页能正常访问素材。

## P5：工程质量和上线准备

### P5.1 拆分生产包大 chunk

目标：解决当前 Vite 生产构建的大 chunk 警告。

任务：

- Monaco Editor 动态加载。
- 编辑器主界面按路由懒加载。
- Ant Design 和图表相关依赖做 manualChunks。
- 发布页运行态尽量避免加载编辑器专用模块。

验收：

- 首屏公共包明显下降。
- 发布页不加载 Monaco 和编辑态设置面板。

### P5.2 CI 完整质量门禁

目标：新提交自动发现构建、测试、类型、后端问题。

任务：

- CI 安装前后端依赖。
- 运行 lint、前端 build、后端 build、Node test。
- 增加 PostgreSQL service，运行 Prisma migrate deploy 和 API smoke。
- 保存 Playwright trace 或测试报告。

验收：

- PR 上能自动显示 check 结果。
- 后端接口破坏权限或保存发布链路时 CI 能失败。

### P5.3 Docker Compose 一键启动

目标：降低本地和测试环境启动成本。

任务：

- 完善 web/server/postgres compose。
- server 容器启动前执行 migration。
- 前端 Nginx 代理 `/api`。
- 补充环境变量说明和排错文档。

验收：

- 新机器能通过文档在 10 分钟内启动完整系统。

### P5.4 安全专项

目标：把低代码平台高风险点显式收口。

任务：

- 自定义 JS 只允许编辑预览，发布页继续禁用。
- HTTP 动作加请求域名白名单或风险提示。
- 上传文件做类型、大小、路径穿越检查。
- JWT secret、数据库密码、对象存储密钥全部通过环境变量管理。
- 增加安全检查清单。

验收：

- 发布页不能执行页面作者写入的任意 JS。
- 上传接口不能写出目标目录。
- 文档能说明当前安全边界和未完成风险。

## P6：数据源与业务集成

### P6.1 数据源管理

目标：让表格、表单、列表不再只依赖静态数据。

当前第一版实现：

- Page 组件属性里可维护 `dataSources` JSON，随页面 schema 保存、版本化和发布。
- 支持 `rest` 和 `static` 两类数据源，REST 请求会复用 HTTP 动作的 API baseURL 和域名白名单策略。
- Table 物料可通过 `dataSourceId` 绑定数据源结果，仍保留 `dataText` 静态数据兜底。
- 数据源结果在运行态以 `context.dataSources.<id>` 暴露给表达式。

任务：

- 新增数据源模型：REST API、静态 JSON、后续数据库查询。
- 支持请求方法、URL、headers、参数、分页字段映射。
- 数据源可绑定到 Table/List/Form Select。
- 数据源请求在预览态和发布态执行。

验收：

- 表格可配置接口并渲染远程数据。
- 请求失败有明确错误提示。

### P6.2 页面变量与表达式

目标：让组件之间可以共享状态和动态计算。

当前第一版实现：

- Page 组件属性里可维护 `variables` JSON，作为运行态页面变量初始值。
- 事件动作新增“设置变量”，可把固定 JSON 值或 `event.value`、`event.values.xxx` 等表达式结果写入变量。
- 组件属性支持 `{{variables.xxx}}` 表达式绑定，运行态渲染前会解析为实际值。
- HTTP 请求动作仍可写入 `event.httpResponse`，并可配合“设置变量”动作把响应关键数据沉淀到页面变量。

任务：

- 引入页面级变量 store。
- 事件动作支持设置变量。
- 组件 props 支持绑定变量或表达式。
- 表单提交结果、HTTP 响应可写入变量。

验收：

- 一个输入框变化可驱动文本、表格查询参数或按钮状态。

建议下一步先做 `P4.1 成员管理前端 UI` 和 `P4.2 审计日志前端 UI`。后端能力已经具备，先把它们产品化，收益最大，也能顺便检验权限模型是否真的顺手。
