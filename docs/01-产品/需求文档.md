# 低代码编辑器需求文档

本文只保留当前产品需求的重点。阶段规划见 `产品路线图.md`，实现细节见架构、接口和编辑器专题文档。

## 产品定位

这是一个面向中后台页面搭建的低代码编辑器。目标不是做万能平台，而是稳定支持用户通过拖拽、配置和发布，完成常见业务页面的搭建、保存、恢复、版本管理和公开访问。

核心闭环：

```text
创建项目 -> 创建页面 -> 拖拽物料 -> 配置属性/样式/事件
  -> 保存 -> 预览 -> 发布 -> 再次编辑 -> 版本回滚
```

## 目标用户

| 用户 | 关注点 |
| --- | --- |
| 前端开发者 | 减少重复页面开发，组件可扩展，schema 可维护 |
| 后台搭建者/实施人员 | 操作直观，页面可保存、可预览、可恢复 |
| 产品/设计 | 快速表达页面结构和基础交互 |
| 管理员 | 管理用户、项目、发布内容和审计记录 |

## 核心价值

- 页面结构用 schema 描述，可保存、复制、版本化和渲染。
- 常见物料可拖拽组合，减少重复 CRUD 页面开发。
- 页面发布和回滚有记录，降低配置错误风险。
- 项目成员权限和管理员后台提供基础治理能力。

## 当前 MVP 范围

### 已包含

- 注册、登录、当前用户读取。
- 项目和页面管理。
- 项目成员 owner/editor/viewer。
- 平台管理员后台。
- 组件拖拽、属性设置、样式设置、事件设置。
- 组件树保存、读取、源码 JSON 应用。
- 版本历史、回滚、删除版本记录。
- 页面发布、取消发布、公开访问。
- schema 迁移和校验。
- API smoke、schema 测试、编辑器 e2e 回归脚本。

### 暂不追求

- 完整商业化低代码平台。
- 多端适配器和复杂页面生成代码。
- 多人实时协同编辑。
- 完整组件市场。
- 自定义 JS 沙箱化后的生产执行。

## 关键功能需求

| 模块 | 需求 |
| --- | --- |
| 用户 | 注册、登录、会话恢复、退出登录 |
| 项目 | 创建、查看、更新、删除、成员管理 |
| 页面 | 创建、打开、保存、删除、路由路径 |
| 编辑器 | 物料区、画布、设置面板、源码面板、大纲树 |
| 物料 | 基础、布局、表单、数据展示、反馈组件 |
| 事件 | 配置动作，支持提示、跳转、HTTP、变量、组件控制等 |
| 版本 | 保存生成版本，支持列表、回滚、删除 |
| 发布 | 生成公开地址，公开页读取发布快照 |
| 管理 | 用户状态、项目状态、发布页治理、全局审计 |
| 安全 | JWT、后端权限校验、公开页禁用 customJS |

## 验收重点

1. 用户能从零创建项目和页面。
2. 页面组件树能保存到后端，并在刷新后恢复。
3. 常用物料在编辑态和预览态表现一致。
4. 属性、样式、事件配置不会破坏 schema。
5. 版本历史可回滚，删除版本不影响当前页面。
6. 发布页读取固定快照，不被草稿自动污染。
7. viewer 不能写页面，非成员不能访问项目。
8. 管理员禁用项目后公开页不可访问。

## 后续需求方向

优先级以 `产品路线图.md` 为准，当前建议顺序：

1. 编辑器 bug 和样式交互修复。
2. 成员管理 UI、审计 UI、版本差异对比。
3. 模板、区块和素材管理。
4. 数据源、变量和表达式联动。
5. 部署演练、监控、错误上报和性能优化。
