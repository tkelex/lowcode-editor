# 后续开发实施计划

## 目标

把当前低代码编辑器从“核心功能可用”推进到“可稳定扩展、可测试、可发布、可长期维护”的企业级项目形态。

后续编码按稳定性优先推进：先保证编辑、预览、保存、发布、回滚这些主链路可靠，再继续扩展物料、数据源和协作能力。

## 推进原则

1. 每个功能都同时考虑编辑态、预览态、保存恢复、发布态和旧数据迁移。
2. 涉及事件、schema、发布页的改动必须补回归验证。
3. UI 问题不只修单个组件，要优先修共用机制，例如画布遮罩、拖拽、运行时状态。
4. 新增物料必须补齐 `dev`、`prod`、`defaultProps`、`setter`、`events`、`methods` 和 `acceptsChildren`。
5. 每个阶段结束运行 `npm.cmd run check`，涉及接口时运行 `npm.cmd run smoke:api`。

## P0：稳定性和回归保障

### 目标

先把低代码编辑器底座打稳，减少“修一个点漏一条链路”的问题。

### 任务清单

- [x] 给事件动作运行时补基础测试：安全条件表达式、动作 URL、HTTP URL、HTTP 请求构造、事件数据提取。
- [x] 补充核心动作执行测试：`toast`、`url`、`confirm`、`condition`、`http` 请求结果、`componentAction`、设置属性/样式。
- [x] 给预览模式补基础回归测试：输入框、选择框、开关、表单、表格、弹窗在预览态能完成服务端渲染，不丢失表单项/表格列等子配置。
- [x] 给发布页补基础回归测试：公开页运行态禁用自定义 JS，事件动作通过注入的 runtime update hooks 更新状态，不直接修改组件快照。
- [x] 建立基础 E2E：通过 Playwright + mock API 覆盖注册登录、创建项目、创建页面、打开编辑器、预览、保存、发布、访问公开页。
- [x] 增加运行日志入口：预览组件渲染异常、事件动作异常可在顶部运行日志中查看。
- [x] 继续完善错误兜底：编辑器预览异常可退出预览或返回项目，发布页加载/迁移/渲染异常可返回首页或刷新页面。
- [x] 明确草稿、编辑预览、发布快照三套状态边界，并补充到架构文档。

### 验收标准

- `npm.cmd run check` 通过。
- 核心链路 E2E 至少覆盖一次完整保存发布流程。
- 预览或发布页出错时用户不会卡死在空白页。

## P1：事件动作系统完善

### 目标

让事件动作从“能配置”升级为“能安全、清晰、可组合地配置”。

### 任务清单

- [x] 维护组件事件能力矩阵：不同组件只展示自己支持的事件，并在 `docs/04-编辑器/事件能力矩阵.md` 记录当前实现。
- [x] 在事件配置 UI 展示事件数据提示，例如 `event.value`、`event.checked`、`event.values`。
- [x] 动作列表支持排序、复制、删除、启用/禁用。
- [x] 嵌套动作配置升级：确认弹窗、条件分支里也能完整添加、编辑、排序、复制、启用/禁用动作列表。
- [x] HTTP 动作支持鉴权选择、headers 模板、body 模板、响应字段映射。
- [x] 条件动作提供可视化配置，减少直接手写表达式。
- [x] 组件联动动作补齐：显示/隐藏、禁用/启用、设置值、清空值、打开/关闭弹窗、提交/重置表单。

### 验收标准

- 所有内置物料事件在文档和注册表中保持一致。
- 动作配置保存后重新打开不丢字段。
- 发布页默认不执行不安全 JS。

## P2：编辑器画布和配置体验

### 目标

提升画布操作手感，让编辑器更接近成熟低代码产品。

### 任务清单

- [x] 优化画布选中态、悬浮态、拖拽态、空状态，参考 amis-editor 的清晰边界和操作按钮。
- [x] 增加响应式视图：桌面、平板、手机宽度切换。
- [x] 右侧配置面板做分组、搜索、折叠和空状态提示。
- [x] 源码区增加格式化、复制、重置、应用前 diff 提示。
- [x] 物料区增加分类、搜索、收藏和常用模板。
- [x] 大纲树支持定位组件、拖拽排序、重命名。
- [x] 增加右键菜单：复制、删除、上移、下移、包裹容器。

### 验收标准

- 画布组件选中框和真实组件尺寸一致。
- 拖拽、选择、删除、配置不产生明显 UI 跳动。
- 移动端宽度下文本和按钮不溢出。

## P3：物料体系扩展

### 目标

补齐搭建常见业务页面所需的基础物料、表单物料和数据展示物料。

### 任务清单

- [x] 基础物料：Link、Icon、Space、Flex、Grid、Tabs、Steps。
- [x] 表单物料：Textarea、Radio、Checkbox、DatePicker、Upload、Rate。
- [x] 数据物料：List、Descriptions、Statistic、Pagination、Chart。
- [x] 反馈物料：Drawer、Tooltip、Popover、Notification、Result、Empty。
- [x] 表格增强：列配置、分页、操作列、空状态、数据源绑定。
- [x] 表单增强：校验规则、默认值、联动、提交、重置、表单数据暴露。
- [x] 页面模板：表单页、详情页、列表页、弹窗表单、仪表盘。

### 验收标准

- 每个新增物料都能编辑、预览、保存、发布。
- 每个新增物料都有事件和动作能力说明。
- 复杂物料至少有一条手测或自动化验证。

## P4：后端和企业化能力

### 目标

让项目从单人本地编辑器扩展到更接近企业实际使用的项目系统。

### 任务清单

- [ ] 权限模型升级：owner、editor、viewer。
- [ ] 项目成员管理：邀请、移除、角色变更。
- [ ] 操作审计：保存、发布、回滚、删除记录操作者和时间。
- [ ] 发布体系升级：草稿版本、发布版本、历史版本差异对比。
- [ ] 页面模板和区块模板后端存储。
- [ ] 素材管理：图片上传、文件存储、资源引用。
- [ ] 后端 DTO、错误码、日志、环境变量继续标准化。

### 验收标准

- 非 owner 不能越权访问项目和页面。
- 发布、回滚、删除都有清晰记录。
- 后端错误响应结构稳定，前端能统一展示。

## P5：工程质量和上线准备

### 目标

建立长期维护所需的工程质量体系。

### 任务清单

- [ ] 引入 Playwright E2E。
- [ ] 增加运行时单元测试，尤其是事件动作和 schema migration。
- [ ] 拆分大 chunk：Monaco、Ant Design、编辑器运行时按需加载。
- [ ] CI：lint、frontend build、server build、test、smoke test。
- [ ] Docker Compose：前端、后端、PostgreSQL 一键启动。
- [ ] 生产部署文档：环境变量、迁移、回滚、日志。
- [ ] 安全检查：自定义 JS、HTTP 动作、发布页、上传能力。

### 验收标准

- 新提交能在 CI 中自动验证。
- 本地和部署启动命令稳定。
- 生产构建没有未解释的关键安全风险。

## 当前建议执行顺序

1. P0 稳定性、P1 事件动作系统、P2 编辑器体验、P3 物料体系已完成，后续改动要继续保持对应回归测试。
2. 优先进入 P4 后端和企业化能力，先从权限模型、项目成员和操作审计开始。
3. P5 工程质量贯穿推进，其中大 chunk 拆分、CI 和部署文档适合在阶段交付前集中收口。
4. 新增物料仍需同步 `dev`、`prod`、配置项、事件能力、schema registry、文档和预览/发布验证。
5. 每个阶段完成后继续更新文档中心和上下文索引，避免计划与代码脱节。
