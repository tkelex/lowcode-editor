# 低代码编辑器后续完善计划

## 规划原则

后续功能按“稳定性优先，然后扩功能”的顺序推进。每次新增能力都要同时考虑：

- 编辑态是否可配置。
- 预览态是否可运行。
- 保存后是否能恢复。
- 发布态是否安全。
- 出错时用户是否能返回或定位问题。
- 旧 schema 是否兼容或可迁移。

## P0：基础稳定性

1. 增加组件 schema 校验器：保存、源码应用、发布前都校验组件树是否合法。
2. 增加全局错误日志面板：预览失败时能看到哪个组件、哪个动作、什么错误。
3. 增加撤销 / 重做：属性、样式、拖拽、事件配置都能回退。
4. 完善旧 schema 迁移：集中按 `schemaVersion` 做 migration，不散落在组件里。
5. 给源码编辑器加格式化、复制、重置、应用前 diff 提示。

## P1：编辑器核心体验

1. 组件复制 / 粘贴 / 删除 / 快捷键。
2. 大纲树支持拖拽排序、重命名、定位组件。
3. 画布支持多选、对齐、间距、层级移动。
4. 右键菜单：复制、删除、上移、下移、包裹容器。
5. 设置面板优化：属性、样式、事件支持搜索和折叠分组。

## P2：事件动作系统

1. 动作编排：动作排序、启用/禁用、失败后是否继续。
2. 动作执行日志：事件触发后能看到每个动作是否成功。
3. HTTP 请求动作支持把返回值写入组件属性。
4. 条件动作支持可视化配置，不只写表达式。
5. 组件联动动作：显示/隐藏、禁用/启用、设置值、清空值。

## P3：物料体系

1. 基础：Tabs、Grid、Flex、Space、Icon、Link。
2. 表单：Radio、Checkbox、DatePicker、Upload、Textarea。
3. 数据：List、Descriptions、Statistic、Chart。
4. 反馈：Drawer、Tooltip、Popover、Notification。
5. 每个物料都补齐：默认 props、setter、stylesSetter、events、methods、dev/prod、是否接收 children。

## P4：数据源和动态页面

1. 增加数据源管理：REST API、静态 JSON、全局变量。
2. 组件属性支持绑定表达式，例如 `{{ api.user.name }}`。
3. 表格、列表、文本支持绑定数据源。
4. 页面级变量和状态管理。
5. 发布页运行时加载数据源。

## P5：发布和项目管理增强

1. 发布记录列表、取消发布、重新发布。
2. 页面模板：保存为模板、从模板创建页面。
3. 资源管理：图片上传、素材库。
4. 项目成员权限：owner/editor/viewer。
5. 页面 SEO、标题、描述、自定义路径。

## P6：工程化和上线准备

1. E2E 测试：拖拽、保存、发布、事件动作、源码应用。
2. CI：lint、build、server build 自动检查。
3. Dockerfile、Nginx、生产部署文档。
4. 错误监控和前端日志上报。
5. 自定义 JS 沙箱化，避免发布页或预览页执行危险代码。
