# 项目架构说明

## 架构概览

项目采用前后端同仓结构：

```text
lowcode-editor/
  src/                 # React 前端和低代码编辑器
  packages/            # 前后端共享类型和 schema 能力
  server/              # NestJS 后端 API
  server/prisma/       # Prisma 数据模型和迁移
  docs/                # 产品、工程、学习和上下文文档
  docker-compose.yml   # 本地 PostgreSQL
```

前端负责编辑器交互和组件树管理，后端负责用户、项目、页面和 schema 持久化，数据库负责保存业务实体和页面 JSON schema。

## 前端架构

### 应用入口

```text
src/main.tsx
  ↓
src/app/providers/AppProviders.tsx
  ↓
src/app/App.tsx
  ↓
登录 / 项目面板 / 编辑器 / 公开发布页四种视图
```

`src/App.tsx` 保留为兼容转发入口，主实现位于 `src/app/App.tsx`。`src/app/App.tsx` 是当前应用级状态切换入口：

- 没有 token：进入 `AuthView`。
- 有 token：加载当前用户，进入 `ProjectDashboard`。
- 打开页面：请求后端 page，先通过共享迁移器规范化 `page.schema`，再取出 `components` 写入 `useComponetsStore`，进入编辑器。
- 访问 `/publish/:publicId`：绕过登录流程，交给 `src/features/publish/PublishedPageView.tsx` 读取公开发布快照并进入只读运行态页面。

应用层已拆分为：

```text
src/app/App.tsx                         # 组合应用流程
src/app/providers/AppProviders.tsx       # 全局 Provider
src/app/hooks/useAuthSession.ts          # 登录态初始化、当前用户、退出登录
src/app/hooks/useEditorPageLoader.ts     # 页面 schema 加载并写入编辑器 store
src/app/components/AppViewOutlet.tsx     # auth/dashboard/editor 视图出口
src/app/components/AppLoading.tsx        # 全局加载态
src/app/routes/publicRoutes.ts           # 公开发布路径识别
src/app/routes/initialView.ts            # 初始视图选择
```

### 编辑器状态

核心文件：`src/editor/stores/components.tsx`

组件节点结构：

```ts
{
  id: number;
  name: string;
  props: any;
  styles?: CSSProperties;
  desc: string;
  children?: Component[];
  parentId?: number;
}
```

核心状态：

- `components`：当前页面组件树。
- `pastComponents` / `futureComponents`：撤销和重做历史，只记录组件树快照，不持久化到 localStorage。
- `curComponentId`：当前选中组件。
- `curComponent`：当前选中组件对象。
- `mode`：编辑或预览。

核心 actions：

- `addComponent`
- `deleteComponent`
- `moveComponent`
- `updateComponentProps`
- `updateComponentStyles`
- `setCurComponentId`
- `setMode`
- `setComponents`
- `undo`
- `redo`

打开页面和版本回滚调用 `setComponents(..., { recordHistory: false })`，作为快照载入并清空历史。源码应用、属性、样式、事件、拖拽和删除会记录历史，可通过 Header 的撤销/重做按钮回退。

组件 store 使用 Zustand persist 保存本地组件树。恢复本地缓存时会通过 `packages/lowcode-schema/src/migrate.ts` 迁移旧 schema，并统一回到编辑模式，避免旧缓存停留在预览态或旧事件字段结构里。

注意：store 导出名是 `useComponetsStore`，拼写错误是现有 API，不要随手改。

### 草稿、编辑预览与发布快照边界

当前页面至少存在三类状态，后续涉及保存、预览、发布和事件动作时必须先判断自己正在修改哪一类状态：

| 状态 | 数据来源 | 渲染入口 | 可执行能力 | 写入位置 |
| --- | --- | --- | --- | --- |
| 编辑草稿 | `useComponetsStore.components`，保存后同步到后端 `Page.schema` | `EditArea` 组件树 | 只提供编辑、拖拽、选中、属性配置 | 编辑器 Zustand store，保存时写 `Page.schema` |
| 编辑预览 | 默认读取 `useComponetsStore.components` | `Preview` 运行态组件树 | 可执行事件动作，默认允许 `custom` 自定义 JS | `setComponentProps` / `setComponentStyles` 写回编辑器 store，方便预览联动 |
| 发布快照 | 公开接口读取 `publishedVersionId` 指向的 `PageVersion.schema` | `PublishedPageView` + `Preview` | 执行安全事件动作，必须传 `allowCustomJS={false}` | 运行态内部快照；组件属性/样式动作只改当前访客页面，不污染编辑器 store |

关键约束：

- 打开页面、源码应用、保存、发布和公开访问都要先经过 `migratePageSchema`，保存和发布前还要经过组件树校验。
- 发布页只读取发布时生成的 `PageVersion.schema`，不会读取正在编辑的 `Page.schema`。用户发布后继续编辑草稿，不会影响公开页面，直到下一次发布。
- `Preview` 收到外部 `components` 参数时进入外部快照模式，会复制一份运行态组件树；事件里的设置属性/样式动作只更新这份运行态状态。
- 编辑预览可以执行自定义 JS，公开发布页不允许执行自定义 JS；条件表达式在公开发布页只能走安全表达式解析。
- 发布页外层有运行时错误兜底，schema 迁移失败、组件渲染异常或运行态异常都应该给出可返回首页或刷新的页面，不能让访客卡在白屏。

### 运行态数据

编辑器预览和公开发布页共用 `src/editor/runtime/Preview/index.tsx`。运行态会从 Page 根组件读取：

- `props.variables`：页面级变量初始值。
- `props.dataSources`：页面级数据源配置。

事件动作可通过 `setVariable` 修改运行态变量。组件属性在渲染前会解析 `{{variables.xxx}}` 表达式，Table 可通过 `dataSourceId` 绑定数据源请求结果。数据源请求和 HTTP 动作共用 `VITE_API_BASE_URL` 与 `VITE_LOWCODE_HTTP_ALLOWED_ORIGINS` 安全边界。

### 组件 schema 迁移与校验

核心文件：

```text
packages/lowcode-schema/src/defaults.ts
packages/lowcode-schema/src/migrate.ts
packages/lowcode-schema/src/validate.ts
packages/lowcode-schema/src/registry.ts
src/editor/schema/validateComponents.ts
server/src/modules/pages/page-schema.service.ts
```

前端在以下入口统一迁移或校验组件树：

- 打开页面写入编辑器 store 前。
- 版本回滚写入编辑器 store 前。
- 公开发布页 Preview 渲染前。
- Zustand 本地缓存恢复时。
- 源码面板应用 JSON 前。
- 保存页面前。
- 发布页面前。

后端在以下入口统一先迁移再校验组件树：

- 创建页面传入初始 schema 时。
- 保存页面 schema 时。
- 发布页面生成发布快照前。
- 回滚页面从 PageVersion 恢复 schema 前。

迁移内容包括：补齐 `schemaVersion`、默认 Page schema、`pageId`、组件 `props/desc` 等基础字段，给异常组件 id 分配稳定递增 id，并把旧事件配置如 `props.onClick.actions`、旧 action type 迁移到 `props.onEvent.click.actions` 的新结构。校验内容包括：根节点数组、非空 Page 根节点、组件 id 有效且唯一、组件 name 已注册、props/styles 基本结构、children 类型、父组件 `acceptsChildren` 规则、已有 `parentId` 与父节点一致性。旧数据如果缺少 `parentId` 暂时兼容。前后端共用 `packages/lowcode-schema`，非法 schema 会在后端返回 `PAGE_SCHEMA_INVALID`。

### 物料注册与渲染

核心文件：`src/editor/registry/component-config.tsx`

`src/editor/stores/component-config.tsx` 仅保留为历史兼容导出，新代码不再从 `stores` 下引入物料注册表。

每个物料通常包含：

```text

src/editor/materials/<Name>/prod.tsx
```

- `dev.tsx`：编辑画布中使用，通常带 `data-component-id` 和拖拽能力。
- `prod.tsx`：预览/运行时使用，负责真实 UI、事件 props 转发和事件执行。
- `component-config.tsx`：注册物料默认 props、分类元信息、搜索关键字、setter、style setter、events、methods、dev/prod 实现。

编辑渲染：`src/editor/components/EditArea/index.tsx`

物料面板：`src/editor/components/Material/index.tsx`

当前物料区从注册表读取可拖拽物料，排除 `Page` 根组件，并按 `layout`、`basic`、`form`、`data`、`feedback` 分类展示。搜索会匹配物料 `name`、中文 `desc` 和 `keywords`。

当前常用物料包括：

```text
Container / Card
Button / Text / Image / Divider
Alert / Modal
Input / Select / Switch / Form / FormItem
Table / TableColumn
```

预览渲染：`src/editor/runtime/Preview/index.tsx`

`src/editor/components/Preview/index.tsx` 和 `src/editor/components/Preivew/index.tsx` 仅保留为历史兼容导出。

事件绑定链路：


### 保存、发布与读取数据流

打开页面：

```text
ProjectDashboard 点击页面
  ↓
src/app/hooks/useEditorPageLoader.ts openPage(pageId)
  ↓
GET /api/pages/:id
  ↓
migratePageSchema(page.schema)
  ↓
setComponents(schema.components)
  ↓
进入 LowcodeEditor
```

保存页面：

```text
Header 点击保存
  ↓
读取 useComponetsStore.components
  ↓
validateComponentTree 校验组件树
  ↓
PATCH /api/pages/:id
  ↓
后端保存到 Page.schema
  ↓
后端创建 PageVersion(source = save)
```

回滚页面：

```text
Header 打开版本历史
  ↓
GET /api/pages/:id/versions
  ↓
选择版本并确认回滚
  ↓
POST /api/pages/:id/rollback
  ↓
后端将 PageVersion.schema 恢复到 Page.schema
  ↓
后端创建 PageVersion(source = rollback)
  ↓
前端 migratePageSchema(page.schema) 后 setComponents(schema.components)
```

删除历史版本：

```text
Header 打开版本历史
  ↓
选择版本并确认删除
  ↓
DELETE /api/pages/:id/versions/:versionId
  ↓
后端删除对应 PageVersion
  ↓
前端刷新版本列表
```

删除历史版本只删除 `PageVersion` 快照，不修改当前 `Page.schema`，因此编辑器画布不会因为删除版本记录而变化。

发布页面：

```text
Header 点击发布
  ↓
先保存并校验当前组件树
  ↓
POST /api/pages/:id/publish
  ↓
后端校验页面所属项目角色，owner/editor 可发布
  ↓
基于当前 Page.schema 创建 PageVersion(source = publish)
  ↓
更新 Page.isPublished / publicId / publishedAt / publishedVersionId
  ↓
前端生成 /publish/:publicId 公开地址
```

公开访问：

```text
访客访问 /publish/:publicId
  ↓
src/app/App.tsx 识别公开路径并跳过登录初始化
  ↓
PublishedPageView 请求公开接口
  ↓
GET /api/public/pages/:publicId
  ↓
后端读取 publishedVersionId 指向的 PageVersion.schema
  ↓
前端 migratePageSchema(page.schema) 后用 Preview 渲染组件树，并传 allowCustomJS = false
```

发布页读取的是发布快照，不是当前可编辑草稿，因此用户发布后继续编辑并保存 `Page.schema` 不会自动影响公开页面，直到再次发布。

### 权限模型

当前已从单用户 owner 模型升级为项目成员角色模型：

```text
User 1 - N Project
Project 1 - N ProjectMember
User 1 - N ProjectMember
Project 1 - N Page
User 1 - N Page(createdBy)
Page 1 - N PageVersion
User 1 - N PageVersion(createdBy)
AuditLog 记录 actorId / projectId / pageId / action
```

权限原则：

- `owner`：项目拥有者，可更新/删除项目、管理成员、查看审计、读写页面、发布/取消发布、回滚和删除版本。
- `editor`：项目编辑者，可读取项目和页面，可创建/保存/发布/取消发布/回滚页面，可删除页面版本。
- `viewer`：项目查看者，只能读取项目、成员、页面和版本，不能写入页面或管理成员。
- 非项目成员不能访问项目和页面；页面接口通过所属项目解析角色。
- 删除 Project 会 cascade 删除 Page。
- 审计日志没有外键级联，删除项目或页面前会先记录操作，日志数据不会因为业务对象删除而被数据库级联删除。

## 数据库架构

核心文件：`server/prisma/schema.prisma`

当前模型：

```text
User
Project
ProjectMember
Page
PageVersion
AuditLog
```

最核心字段：

```prisma
Page.schema Json
Page.publicId String?
Page.isPublished Boolean
Page.publishedAt DateTime?
Page.publishedVersionId Int?
```

它保存低代码编辑器的页面草稿 schema，在 PostgreSQL 中对应 JSONB。发布相关字段记录公开访问 ID、发布状态、发布时间和当前公开快照版本。

历史版本字段：

```prisma
PageVersion.schema Json
```

保存页面会生成 `source = "save"` 的版本；回滚页面会生成 `source = "rollback"` 的版本；发布页面会生成 `source = "publish"` 的公开访问快照。

项目成员字段：

```prisma
ProjectMember.projectId Int
ProjectMember.userId Int
ProjectMember.role ProjectMemberRole
```

`ProjectMemberRole` 当前为 `OWNER`、`EDITOR`、`VIEWER`。API 层返回小写 `owner/editor/viewer`，避免前端直接依赖数据库枚举命名。

审计日志字段：

```prisma
AuditLog.actorId Int?
AuditLog.projectId Int?
AuditLog.pageId Int?
AuditLog.action String
AuditLog.targetType String
AuditLog.targetId String?
AuditLog.metadata Json?
```

审计动作当前覆盖项目创建/更新/删除、成员新增/变更/移除、页面创建/保存/发布/取消发布/回滚/删除和版本删除。

## 高风险修改区域

- `src/editor/stores/components.tsx`：影响整个编辑器组件树。
- `src/editor/registry/component-config.tsx`：影响物料注册和属性面板。
- `src/editor/runtime/Preview/index.tsx`：执行事件动作，包括编辑器预览中的 `customJS`；公开发布页必须传 `allowCustomJS={false}`。
- `src/features/publish/PublishedPageView.tsx`：公开发布页入口，必须保持 `allowCustomJS={false}`。
- `server/src/modules/projects/project-access.service.ts`：项目角色权限解析，影响所有项目和页面 API。
- `server/src/modules/projects/projects.service.ts`：项目成员管理、项目设置和审计查询。
- `server/src/modules/pages/pages.service.ts`：页面角色权限和页面 CRUD 编排。
- `server/src/modules/audit/audit-logs.service.ts`：关键操作审计写入。
- `server/src/modules/pages/page-versions.service.ts`：版本号、回滚和删除历史版本。
- `server/src/modules/pages/page-publish.service.ts`：发布快照和公开读取。
- `server/src/modules/pages/page-schema.service.ts`：schema normalize。
- `server/prisma/schema.prisma`：数据库结构变更必须配套 migration。
