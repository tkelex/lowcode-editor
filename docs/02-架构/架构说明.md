# 项目架构说明

本文只保留当前有效架构和关键数据流。文件级职责详见 `.claude/context/FILE_MAP.md`，接口细节详见 `docs/03-接口/接口说明.md`。

## 总体结构

```text
lowcode-editor/
  src/                  React 前端和低代码编辑器
  packages/lowcode-schema/ 前后端共享 schema 类型、迁移、校验
  server/               NestJS API
  server/prisma/        Prisma schema 和 migrations
  docs/                 项目文档
  infra/                Docker / Nginx 示例
```

前端负责编辑体验、项目后台和运行态预览；共享包负责 schema 规则、数据源模型校验和 CRUD schema 生成；后端负责鉴权、项目权限、页面持久化、数据源模型配置、版本和发布；数据库保存平台业务实体和 JSON schema，但不托管用户自己的业务记录数据。

## 前端架构

入口链路：

```text
src/main.tsx
  -> src/app/providers/AppProviders.tsx
  -> src/app/App.tsx
  -> AuthView / ProjectDashboard / LowcodeEditor / PublishedPageView
```

主要职责：

- `src/app/App.tsx`：应用状态和视图分发。
- `useAuthSession`：登录态初始化、当前用户、退出登录。
- `useEditorPageLoader`：打开页面，读取后端 schema，迁移后写入编辑器 store。
- `ProjectDashboard`：项目、页面、数据源模型和 CRUD 生成入口。
- `PublishedPageView`：公开发布页，读取发布快照并禁用 customJS。

## 编辑器架构

编辑器入口：`src/editor/index.tsx`。

编辑模式：

```text
Header
Left: Material / AI / Outline / Source
Center: EditArea
Right: Setting
```

预览模式：

```text
Header
Preview runtime component tree
```

核心模块：

- `stores/components.tsx`：组件树、当前选中、编辑模式、撤销重做。
- `registry/component-config.tsx`：物料元信息、默认 props、setter、事件、方法、dev/prod 实现。
- `components/EditArea`：递归渲染 dev 物料，处理选择、hover、右键、画布。
- `runtime/Preview`：递归渲染 prod 物料，注入事件动作、变量、数据源。
- `components/Setting`：属性、样式、事件设置。
- `components/Material`：物料分类、搜索、收藏、模板。
- `components/AiBuilderPanel`：AI 页面搭建和 agent 修改入口；生成/修改结果先预览和确认，再写入组件树。
- `hooks/useMaterialDrop.ts`：新增组件和移动组件的拖拽逻辑。

## 组件树模型

```ts
{
  id: number;
  name: string;
  props: any;
  styles?: CSSProperties;
  desc: string;
  children?: Component[];
  parentId?: number;
}
```

约定：

- 根组件是 `Page`。
- 历史导出名 `useComponetsStore` 保持兼容。
- 组件树修改会记录 undo/redo 快照；历史栈不持久化。
- 打开页面、回滚、缓存恢复会先迁移 schema，再写入 store。

## Schema 架构

共享包位置：`packages/lowcode-schema/src`。

| 文件 | 职责 |
| --- | --- |
| `types.ts` | 页面、组件、事件 action 类型 |
| `defaults.ts` | 当前 schema 版本和默认 Page |
| `migrate.ts` | 旧 schema 迁移 |
| `validate.ts` | 组件树校验 |
| `registry.ts` | 物料父子关系 |
| `data-model-crud-*` | 前端数据源模型类型、校验和 CRUD 页面 schema 生成 |
| `url.ts`、`http-action.ts`、`safe-expression.ts` | 运行态安全能力 |
| `ai-types.ts`、`ai-registry.ts`、`ai-normalize.ts`、`ai-validate.ts` | AI 页面生成协议、物料白名单、规范化和写入前校验 |
| `ai-agent-types.ts`、`ai-agent-patch.ts` | AI agent run、上下文、工具调用、候选 patch、stale baseline 和 patch 应用校验 |

统一规则：

- 前端打开、源码应用、保存、发布前都要迁移或校验。
- 后端创建、保存、发布、回滚前都要迁移和校验。
- 旧事件字段迁移到 `props.onEvent[eventName].actions`。

## 运行态边界

| 场景 | 数据来源 | 渲染入口 | 能力 |
| --- | --- | --- | --- |
| 编辑态 | Zustand components | `EditArea` | 选择、拖拽、配置 |
| 编辑器预览 | Zustand components | `Preview` | 事件动作，默认允许 customJS |
| 公开发布页 | PageVersion publish 快照 | `PublishedPageView` + `Preview` | 禁止 customJS |

公开页必须保持：

```tsx
<Preview components={components} allowCustomJS={false} />
```

## 后端架构

入口：

```text
server/src/main.ts
  -> server/src/app.module.ts
  -> auth / users / projects / pages / data-source-models / admin / audit / assets / templates
```

核心模块：

- `auth`：注册、登录、JWT。
- `projects`：项目 CRUD、成员管理、权限解析、项目审计。
- `pages`：页面 CRUD、schema normalize、版本、发布。
- `data-source-models`：项目级外部 API 数据源模型配置 CRUD，只保存配置，不保存外部业务记录。
- `admin`：平台管理员后台 API。
- `ai`：AI 页面生成和 agent 编排网关，负责模型调用、白名单工具、权限校验、审计、降级草稿生成和候选 patch 校验。
- `audit`：审计日志写入和查询。
- `prisma`：数据库访问。

## 权限模型

项目角色：

- `owner`：项目管理、成员管理、页面读写、发布、回滚、审计、数据源模型维护。
- `editor`：页面读写、发布、回滚、数据源模型维护。
- `viewer`：只读项目、页面、版本和数据源模型。

管理员：

- 用户 `role=admin` 且 `status=active`。
- 由 `AdminGuard` 从数据库复查，不能只依赖前端或旧 JWT。

## 数据库模型

核心模型：

```text
User
Project
ProjectMember
Page
PageVersion
AuditLog
Asset
Template
ProjectDataSourceModel
```

页面相关字段：

- `Page.schema`：当前编辑草稿。
- `PageVersion.schema`：保存、回滚、发布生成的快照。
- `Page.publicId`：公开访问 ID。
- `Page.publishedVersionId`：当前公开快照。

数据源模型字段：

- `ProjectDataSourceModel.fields`：字段映射 JSON，用于生成 TableColumn、FormItem 和详情展示。
- `listApi/detailApi/createApi/updateApi/deleteApi`：外部 REST API 配置 JSON。
- 这些配置只服务前端低代码页面生成和运行态请求，不代表平台托管了外部业务记录。

## 数据源模型和 CRUD 生成数据流

```text
ProjectDashboard 数据源模型
  -> POST/PATCH /api/projects/:projectId/data-source-models
  -> 后端复用 ProjectAccessService 校验 owner/editor
  -> 保存外部 API 配置和字段映射
  -> 前端 generateCrudPageSchema
  -> POST /api/projects/:projectId/pages
  -> 生成普通 Page schema，后续继续走保存、版本和发布链路
```

生成页约束：

- 列表读取优先使用 Page `dataSources` 和 Table `dataSourceId`。
- 新增、编辑等写入动作使用 `props.onEvent[eventName].actions` 中的 HTTP action。
- 公开发布页仍禁用 custom JS，并继续遵守 HTTP allowed origins。

## 保存和发布数据流

保存：

```text
Header 保存
  -> validateComponentTree
  -> PATCH /api/pages/:id
  -> 后端 migrate + validate
  -> 更新 Page.schema
  -> 创建 PageVersion(source=save)
```

发布：

```text
Header 发布
  -> 先保存当前组件树
  -> POST /api/pages/:id/publish
  -> 基于 Page.schema 创建 PageVersion(source=publish)
  -> 更新 publicId / isPublished / publishedVersionId
```

公开访问：

```text
/publish/:publicId
  -> GET /api/public/pages/:publicId
  -> 读取 publishedVersionId 指向的 PageVersion.schema
  -> 前端迁移后 Preview 渲染
```

## AI 页面搭建数据流

```text
编辑器 AI 页签
  -> POST /api/pages/:id/ai/page-generation 或 /api/projects/:id/ai/page-generation
  -> 后端校验 owner/editor 权限
  -> 有模型配置时调用 OpenAI 兼容 chat completions
  -> 无模型配置或模型结果不可用时生成本地规则草稿
  -> validateAiGeneratedComponents 校验物料、父子关系、事件动作和 customJS 边界
  -> 前端展示摘要、warnings、assumptions 和预览
  -> 用户确认后替换整页或插入当前容器
  -> 继续走现有保存、版本和发布流程
```

AI 只生成低代码 schema，不生成任意应用源码；模型密钥只存在于后端环境变量中。

## AI Agent 编排数据流

```text
编辑器 AI 页签 Agent 修改
  -> POST /api/pages/:id/ai/agent-runs 或 /api/projects/:id/ai/agent-runs
  -> 后端校验 owner/editor 权限
  -> 构建受限上下文包：当前组件树、选中组件、物料能力、数据源/接口说明摘要
  -> 按白名单工具执行 read/generate/patch/validate 步骤
  -> 输出候选 patch 或候选组件树、执行轨迹、warnings 和 assumptions
  -> 前端展示 diff/预览，用户确认后应用到 Zustand 组件树
  -> 继续走现有保存、版本和发布流程
```

Agent 工具不能直接保存页面、发布页面、执行 shell、访问任意外部系统或生成默认 `custom` action。候选 patch 应用时会重新校验当前组件树基准，防止旧候选覆盖用户后续手动修改。

## 当前架构风险

- 编辑器 store 是中心状态，改动影响面大。
- registry 和共享 schema registry 必须保持一致。
- Preview 同时服务编辑器预览和公开页，customJS 边界必须清楚。
- 权限不能只在前端处理。
- schema 校验过严会拦截旧页面，过松会保存非法组件树。
