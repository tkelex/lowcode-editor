# 项目架构说明

本文只保留当前有效架构和关键数据流。文件级职责详见 `.claude/context/FILE_MAP.md`，接口细节详见 `docs/03-接口/接口说明.md`。

## 总体结构

```text
lowcode-editor/
  src/                  React 前端和低代码编辑器
  packages/lowcode-schema/ 前后端共享 schema 类型、迁移、校验
  server/               NestJS API
  server/prisma/        Prisma schema 和 migrations
  docs/                 项目文档
  infra/                Docker / Nginx 示例
```

前端负责编辑体验和运行态预览；共享包负责 schema 规则；后端负责鉴权、项目权限、页面持久化、版本和发布；数据库保存业务实体和 JSON schema。

## 前端架构

入口链路：

```text
src/main.tsx
  -> src/app/providers/AppProviders.tsx
  -> src/app/App.tsx
  -> AuthView / ProjectDashboard / LowcodeEditor / PublishedPageView
```

主要职责：

- `src/app/App.tsx`：应用状态和视图分发。
- `useAuthSession`：登录态初始化、当前用户、退出登录。
- `useEditorPageLoader`：打开页面，读取后端 schema，迁移后写入编辑器 store。
- `ProjectDashboard`：项目和页面入口。
- `PublishedPageView`：公开发布页，读取发布快照并禁用 customJS。

## 编辑器架构

编辑器入口：`src/editor/index.tsx`。

编辑模式：

```text
Header
Left: Material / Outline / Source
Center: EditArea
Right: Setting
```

预览模式：

```text
Header
Preview runtime component tree
```

核心模块：

- `stores/components.tsx`：组件树、当前选中、编辑模式、撤销重做。
- `registry/component-config.tsx`：物料元信息、默认 props、setter、事件、方法、dev/prod 实现。
- `components/EditArea`：递归渲染 dev 物料，处理选择、hover、右键、画布。
- `runtime/Preview`：递归渲染 prod 物料，注入事件动作、变量、数据源。
- `components/Setting`：属性、样式、事件设置。
- `components/Material`：物料分类、搜索、收藏、模板。
- `hooks/useMaterialDrop.ts`：新增组件和移动组件的拖拽逻辑。

## 组件树模型

```ts
{
  id: number;
  name: string;
  props: any;
  styles?: CSSProperties;
  desc: string;
  children?: Component[];
  parentId?: number;
}
```

约定：

- 根组件是 `Page`。
- 历史导出名 `useComponetsStore` 保持兼容。
- 组件树修改会记录 undo/redo 快照；历史栈不持久化。
- 打开页面、回滚、缓存恢复会先迁移 schema，再写入 store。

## Schema 架构

共享包位置：`packages/lowcode-schema/src`。

| 文件 | 职责 |
| --- | --- |
| `types.ts` | 页面、组件、事件 action 类型 |
| `defaults.ts` | 当前 schema 版本和默认 Page |
| `migrate.ts` | 旧 schema 迁移 |
| `validate.ts` | 组件树校验 |
| `registry.ts` | 物料父子关系 |
| `url.ts`、`http-action.ts`、`safe-expression.ts` | 运行态安全能力 |

统一规则：

- 前端打开、源码应用、保存、发布前都要迁移或校验。
- 后端创建、保存、发布、回滚前都要迁移和校验。
- 旧事件字段迁移到 `props.onEvent[eventName].actions`。

## 运行态边界

| 场景 | 数据来源 | 渲染入口 | 能力 |
| --- | --- | --- | --- |
| 编辑态 | Zustand components | `EditArea` | 选择、拖拽、配置 |
| 编辑器预览 | Zustand components | `Preview` | 事件动作，默认允许 customJS |
| 公开发布页 | PageVersion publish 快照 | `PublishedPageView` + `Preview` | 禁止 customJS |

公开页必须保持：

```tsx
<Preview components={components} allowCustomJS={false} />
```

## 后端架构

入口：

```text
server/src/main.ts
  -> server/src/app.module.ts
  -> auth / users / projects / pages / admin / audit / assets / templates
```

核心模块：

- `auth`：注册、登录、JWT。
- `projects`：项目 CRUD、成员管理、权限解析、项目审计。
- `pages`：页面 CRUD、schema normalize、版本、发布。
- `admin`：平台管理员后台 API。
- `audit`：审计日志写入和查询。
- `prisma`：数据库访问。

## 权限模型

项目角色：

- `owner`：项目管理、成员管理、页面读写、发布、回滚、审计。
- `editor`：页面读写、发布、回滚。
- `viewer`：只读项目、页面和版本。

管理员：

- 用户 `role=admin` 且 `status=active`。
- 由 `AdminGuard` 从数据库复查，不能只依赖前端或旧 JWT。

## 数据库模型

核心模型：

```text
User
Project
ProjectMember
Page
PageVersion
AuditLog
Asset
Template
```

页面相关字段：

- `Page.schema`：当前编辑草稿。
- `PageVersion.schema`：保存、回滚、发布生成的快照。
- `Page.publicId`：公开访问 ID。
- `Page.publishedVersionId`：当前公开快照。

## 保存和发布数据流

保存：

```text
Header 保存
  -> validateComponentTree
  -> PATCH /api/pages/:id
  -> 后端 migrate + validate
  -> 更新 Page.schema
  -> 创建 PageVersion(source=save)
```

发布：

```text
Header 发布
  -> 先保存当前组件树
  -> POST /api/pages/:id/publish
  -> 基于 Page.schema 创建 PageVersion(source=publish)
  -> 更新 publicId / isPublished / publishedVersionId
```

公开访问：

```text
/publish/:publicId
  -> GET /api/public/pages/:publicId
  -> 读取 publishedVersionId 指向的 PageVersion.schema
  -> 前端迁移后 Preview 渲染
```

## 当前架构风险

- 编辑器 store 是中心状态，改动影响面大。
- registry 和共享 schema registry 必须保持一致。
- Preview 同时服务编辑器预览和公开页，customJS 边界必须清楚。
- 权限不能只在前端处理。
- schema 校验过严会拦截旧页面，过松会保存非法组件树。
