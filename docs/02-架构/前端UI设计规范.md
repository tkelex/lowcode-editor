# 前端 UI 设计规范

本文档用于约束本项目后续界面开发，尤其是编辑器三栏布局、物料区、画布区、右侧设置区和管理后台。

## 设计目标

- 编辑器界面优先像“专业生产工具”，不要像营销页或后台表单堆叠。
- 信息密度要高，但必须可扫读：标题、配置项、操作按钮各自占清晰位置。
- 默认避免大圆角、大阴影、大面积彩色背景、渐变装饰和过度卡片化。
- 页面不能出现非必要横向滚动。固定面板内文字必须截断或换行，不允许挤压布局。

## 编辑器布局

- 顶部栏：只放全局操作，按钮文案短，状态信息弱化。
- 左侧物料区：使用轻量分组和紧凑列表，素材卡片不做厚重阴影。
- 中间画布：保持可编辑区域清晰，选中态使用蓝色描边，不用复杂装饰。
- 右侧设置区：参考 amis editor 的侧栏气质，使用“顶部 Tab + 灰色分组条 + 白色内容”的结构。

## 右侧设置区

- 顶部只能保留 `属性 / 样式 / 事件` 切换，不展示 `Container #123` 这类内部组件名和 ID。
- 搜索框作为辅助工具，不能抢占视觉层级。
- 配置分组使用浅灰标题条，内容区保持白底，不把每个分组做成厚重卡片。
- 表单项优先横向 label + 控件，label 宽度稳定，避免文字挤成两排。
- 事件区只展示必要信息：事件名、动作数量、事件数据和动作列表。
- 动作按钮使用图标按钮，默认弱化，hover 时再突出。

## 字体与颜色

- 主文本使用接近黑色但不使用纯黑：`#151b26`。
- 次级文本使用灰蓝：`#4f5b6b`、`#7a8492`。
- 边框使用低对比灰：`#e6e8eb`、`#eef0f3`。
- 主强调色只使用一类蓝色：`#2468f2`。
- 禁止在编辑器工具界面使用大面积紫色、蓝紫渐变、装饰光斑。

## 自动检查清单

每次改编辑器 UI 后至少检查：

- `npm run build`
- `npm run test:e2e -- e2e/editor-regression.spec.ts`
- 右侧设置区不出现内部组件 ID。
- `.setting-panel`、`.edit-area` 不产生横向滚动。
- 预览模式能正常显示输入类组件，并且可以退出预览。

## AI 编码要求

- 写 UI 前先判断当前界面属于工具型、管理型还是展示型，不能套用同一种卡片风格。
- 对工具型界面，优先控制密度、边界、对齐和可扫读性。
- 修改后必须用截图或 E2E 检查验证，不只凭代码判断。
