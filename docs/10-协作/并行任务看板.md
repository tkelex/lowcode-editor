# 并行任务看板

这个文件用于多个 AI 对话或协作者同时工作时避免互相覆盖。每个新窗口开始前都应先读本文件和 `git status --short`。

## 使用规则

1. 一个窗口只负责一个明确任务。
2. 每个任务必须声明“允许修改”和“禁止修改”的文件范围。
3. 高风险文件同一时间只能分配给一个窗口。
4. 如果目标文件已经有未提交改动，先报告，不要直接覆盖。
5. 禁止在普通任务里运行全仓库格式化或批量自动修复。
6. 任务完成后，把状态改为“待合并”或“已完成”，并记录验证命令。

## 推荐并行组合

适合并行：

- 一个窗口做只读审查。
- 一个窗口修独立 UI 样式。
- 一个窗口改后端某个独立 service。
- 一个窗口补文档或测试。

不适合并行：

- 多个窗口同时改编辑器核心 store。
- 多个窗口同时改物料 registry。
- 一个窗口改 schema，另一个窗口改保存/发布链路。
- 多个窗口同时做大范围“优化编辑器”。

## Git Worktree 建议

如果需要更强隔离，可以给不同任务使用不同 worktree：

```bash
git worktree add ../lowcode-editor-docs codex/docs-cleanup
git worktree add ../lowcode-editor-ui codex/editor-ui-fix
git worktree add ../lowcode-editor-api codex/api-fix
```

即使用 worktree，也建议保留本看板，因为逻辑资源仍可能冲突。
