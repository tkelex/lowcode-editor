# 项目实施计划

## 目标

把当前低代码编辑器前端原型逐步完善为可上线的低代码平台 MVP。第一阶段优先完成登录、项目、页面、schema 保存和读取闭环。

## 阶段划分

### Phase 0：工程基线

- 初始化 git。
- 补齐 `.gitignore`。
- 建立编码、安全、Git 工作流文档。
- 保留现有前端结构，避免第一阶段大规模移动文件。

验收标准：

- git 仓库可用。
- 敏感文件不会被追踪。
- 文档可指导后续实现。

### Phase 1：后端基础

- 在 `server/` 新增 NestJS API 服务。
- 接入 Prisma 和 PostgreSQL。
- 建立 `User`、`Project`、`Page` 初始模型。
- 提供 `.env.example`。

验收标准：

- 后端可以启动和构建。
- Prisma Client 可以生成。
- 数据库连接通过环境变量配置。

### Phase 2：认证

- 实现注册、登录、当前用户接口。
- 密码哈希保存。
- JWT 保护业务接口。

验收标准：

- 用户可以注册和登录。
- 无 token 不能访问受保护接口。
- `GET /auth/me` 返回当前用户。

### Phase 3：项目与页面

- 实现项目 CRUD。
- 实现页面 CRUD。
- 页面 schema 保存到数据库。
- 页面读取时返回 schema。

验收标准：

- 登录用户可以创建项目和页面。
- 用户只能访问自己的数据。
- 页面 schema 可保存和读取。

### Phase 4：前端闭环

- 新增 API 层。
- 新增登录/注册入口。
- 新增项目列表和页面列表。
- 编辑器打开页面时加载 schema。
- 保存按钮将当前组件树提交后端。

验收标准：

- 登录后能创建项目和页面。
- 编辑器保存后刷新不丢失。
- 预览模式不回归。

### Phase 5：验证与上线准备

- 前端 lint/build 通过。
- 后端 build 通过。
- 手动验证注册、登录、项目、页面、保存、刷新恢复。
- 后续补 Docker、部署、版本和发布能力。

## 第一阶段不做

- 多人实时协作。
- 复杂成员权限。
- 组件市场。
- 自定义代码组件上传。
- 多环境发布审批。
- 完整 CI/CD。

这些能力等保存闭环稳定后再做。
