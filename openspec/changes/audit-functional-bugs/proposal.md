## Why

当前项目经历了多轮样式与交互改造，部分界面可能已经具备视觉表现，但实际功能接线、状态更新、预览运行时行为或异常路径仍可能存在缺口。需要进行一次面向真实可用性的代码审查，系统找出“看起来可用但没有实际作用”的控件、潜在运行时错误和编辑器核心流程中的隐藏 bug。

## What Changes

- 新增一次功能有效性审查，覆盖编辑器编辑态、预览态、物料拖拽、属性/样式/事件配置、项目仪表盘、源码/大纲等主要工作流。
- 输出可复现的问题清单，标注文件位置、触发步骤、用户影响、风险等级和建议修复方向。
- 对发现的问题建立后续修复任务，优先处理无实际作用的按钮/表单项、状态未写入、预览与编辑态不一致、事件动作未生效、保存/加载链路断裂等问题。
- 补充或更新必要的验证方式，确保修复后的功能有 lint/test/build 或人工验收路径。

## Capabilities

### New Capabilities

- `functional-bug-audit`: 规定项目功能有效性审查的覆盖范围、问题记录标准和验收要求。

### Modified Capabilities

无。

## Impact

- 主要影响 `src/editor/**`、`src/app/**`、`src/pages/**`、`src/services/**`、`server/**`、`e2e/**` 与相关测试/文档。
- 不引入新的运行时依赖；若审查发现缺少测试夹具或验证脚本，应优先复用现有 `npm run lint`、`npm run test`、`npm run build`、`npm run check`。
- 审查本身不改变公开 API，但后续修复可能触及组件树 store、物料注册表、事件动作运行时和项目持久化接口。
