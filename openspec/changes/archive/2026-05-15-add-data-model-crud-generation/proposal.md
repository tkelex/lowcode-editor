## Why

当前项目的目标应收敛为“前端低代码平台”，核心价值是页面搭建、物料体系、数据源绑定、事件编排、预览和发布，而不是 JeecgBoot 那类覆盖后端、数据库、工作流和代码生成的全栈企业开发平台。

因此，本次变更从“平台托管业务数据模型”调整为“前端数据源模型 + CRUD 页面生成”：用户配置外部 API、字段映射和页面生成选项，系统生成可编辑、可预览、可发布的前端 CRUD 页面 schema。

## What Changes

- 新增项目级前端数据源模型配置，描述外部 API、认证方式、请求方法、响应路径、主键字段和字段映射。
- 新增 CRUD 页面生成入口，从数据源模型生成列表页、表单页和详情页的低代码 schema。
- 生成页面继续使用现有组件树、物料 registry、事件动作、运行时数据源、保存、版本和发布机制。
- 不新增平台托管业务记录表，不提供通用业务数据 CRUD 存储接口，不生成后端数据库表。
- 增强事件动作和数据源能力，使生成页面可以调用外部新增、编辑、删除、查询接口，并展示请求结果或错误。
- 新增前端数据源模型管理 UI、生成向导、schema 生成器和验证覆盖。

## Capabilities

### New Capabilities

- `data-model-crud-generation`: 定义前端数据源模型，并基于模型生成可编辑、可运行、可发布的 CRUD 页面 schema。

### Modified Capabilities

- `event-actions`: CRUD 生成页面需要通过现有事件动作触发表单提交、删除确认、列表刷新、详情跳转和外部 API 调用，并要求动作选择与生成组件能力保持一致。
- `editor-interaction-styling`: 数据源模型管理、CRUD 生成向导和生成页面必须保持现有编辑器/后台的可读、可搜索和可验证交互标准。

## Impact

- 前端：项目后台入口、数据源模型管理页面、CRUD 生成向导、shared api 封装、编辑器 schema 写入、必要的物料/事件配置增强。
- 共享 schema：前端数据源模型类型、字段映射类型、CRUD 生成器、schema 校验和测试工具。
- 后端：只保存项目级数据源模型配置，复用项目权限和审计；不托管业务记录数据。
- 文档：接口文档、架构文档、上下文索引需说明“前端低代码平台”定位和数据源模型边界。
- 验证：`npm run lint`、`npm run build`、`npm run test`、后端 build、关键 e2e 编辑器/生成流程。
