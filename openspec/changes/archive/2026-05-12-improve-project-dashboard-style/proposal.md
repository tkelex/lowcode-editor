## Why

项目工作台是用户进入低代码编辑器前的主要入口，但当前页面留白过大、信息层级偏弱，项目列表和页面列表缺少足够的工作台质感。需要在不改变业务流程的前提下，让页面更易扫描、更稳重，也更符合现有编辑器产品的视觉标准。

## What Changes

- 优化项目工作台的整体背景、页头、操作区、左右面板间距和响应式布局。
- 强化项目列表的选中态、角色标签、描述文本和空状态，让用户更快判断当前项目。
- 优化页面列表的信息密度、发布状态、路径、更新时间和行级操作呈现。
- 保持现有项目、成员、审计、发布记录、新建项目和新建页面流程不变。
- 不引入新的后端接口、依赖或数据模型变更。

## Capabilities

### New Capabilities

- `project-dashboard-style`: 定义项目工作台在项目选择、页面管理、权限状态和响应式布局下应具备的视觉层级与交互反馈。

### Modified Capabilities

- 无。

## Impact

- 主要影响前端项目工作台：
  - `src/features/projects/ProjectDashboard.tsx`
- 可能影响的验证：
  - `npm run lint`
  - `npm run build`
- 不影响后端 API、数据库 schema、鉴权逻辑、编辑器组件树 schema 或发布页面运行时。
