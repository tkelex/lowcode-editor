## ADDED Requirements

### Requirement: 项目工作台应提供清晰的页面层级

项目工作台 MUST 以工作台式布局呈现标题、当前用户、主操作、项目列表和页面列表，并通过中性背景、白色面板、细边框、间距和标题层级区分主要区域。

#### Scenario: 用户打开项目工作台

- **WHEN** 项目工作台加载完成
- **THEN** 页面应显示清晰的页头、当前用户信息和主要操作按钮
- **AND** 项目列表与页面列表应作为两个独立工作面板呈现
- **AND** 视觉风格应保持克制的 SaaS 操作界面，不使用营销化大 hero 或装饰性大渐变

### Requirement: 项目列表应强化当前选择状态

项目列表 MUST 让用户能够快速识别当前选中的项目、项目角色和项目描述。

#### Scenario: 当前项目被选中

- **WHEN** 用户选中某个项目
- **THEN** 该项目行应有明确的背景、边框或文字层级反馈
- **AND** 项目角色应以语义标签展示
- **AND** 项目描述为空时应显示低对比度占位文本

### Requirement: 页面列表应提升扫描效率

页面列表 MUST 以稳定的行布局展示页面名称、发布状态、路径、更新时间和行级操作。

#### Scenario: 项目存在多个页面

- **WHEN** 当前项目下存在页面
- **THEN** 每个页面应以可扫描行展示页面名称和状态
- **AND** 页面路径与更新时间应作为辅助信息展示
- **AND** 发布记录与打开编辑器操作应在行尾保持清晰可点击

### Requirement: 项目工作台应支持小屏布局

项目工作台 MUST 在窄视口下保持内容可读且不出现明显横向溢出。

#### Scenario: 用户在较窄视口查看工作台

- **WHEN** 视口宽度不足以显示左右两栏
- **THEN** 项目列表和页面列表应堆叠展示
- **AND** 页头操作、卡片标题操作和列表行操作应允许换行
